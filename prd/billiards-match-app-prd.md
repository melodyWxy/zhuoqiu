## 1. 产品概述

### 1.1 项目背景

随着桌球运动在国内的普及，业余球友对专业计分工具的需求日益增长。传统纸质计分方式存在记录不便、数据难以追溯、赛事管理效率低等问题。本项目旨在打造一款专为桌球爱好者设计的智能计分与管理平台。

### 1.2 产品定位

一款面向业余桌球爱好者的**极简计分工具**，让球友在球房打球时**不再依赖计分牌和扑克牌**，操作简单到拿起手机就能用。

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| 告别计分牌 | 一键记录胜负，再也不用记分牌和扑克牌 |
| 操作极简 | 进球点一下，犯规点一下，结束看一眼 |
| 八球只记输赢 | 中式八球只关注谁赢了这一局 |
| 九球记精彩 | 九球记录普胜、黄金9、大小金，记住你的高光时刻 |
| 多人赛事 | 支持2人、3人同时竞技 |

### 1.4 名词定义

| 术语 | 定义 |
|------|------|
| 中式八球 | 中国特色8球台球，使用1-7号为全色球，9-15号为花色球，8号为黑球 |
| 九球 | 使用1-9号球，玩家需按顺序击打最低号码球 |
| 普胜 | 九球追分赛中，普通单局胜利（清完部分球型后打进9号球），得2分 |
| 小金 | 九球追分赛中，开球进球后传进9号球/清台(无进球)/开球直接进9号球，得4分 |
| 大金 | 九球追分赛中，开球方完成开球清台（打进1-9号全部球），得6分 |
| 三犯 | 九球追分赛中，迫使对方连续犯规3次，得1分 |
| 胜方开球 | 九球追分赛规则，每局由获胜方开球 |
| 球友圈 | 应用内球友社交功能模块 |
| 球房 | 桌球房/台球俱乐部的简称 |

## 2. 用户分析

### 2.1 目标用户画像

#### 用户类型 A：普通球友
- **年龄：** 18-45岁
- **特征：** 业余爱好者，每周打球1-3次
- **需求：** 便捷计分、记录战绩、查看历史记录
- **痛点：** 打球时用手机或纸笔记录不方便

#### 用户类型 B：资深玩家
- **年龄：** 25-50岁
- **特征：** 经常参与球房活动，有固定球友圈
- **需求：** 专业赛事计分、战绩统计、排名展示
- **痛点：** 缺少专业工具，现有工具不够精准

#### 用户类型 C：球房老板/赛事组织者
- **年龄：** 30-55岁
- **特征：** 经营球房或经常组织比赛
- **需求：** 便捷办赛、吸引会员、提升球房知名度
- **痛点：** 办赛流程繁琐，用户参与门槛高

### 2.2 用户增长策略

```
球房获客 → 种子用户培养 → 口碑传播 → 球友圈裂变
```

## 3. 功能需求

### 3.1 一期功能（MVP）

#### 3.1.1 计分核心功能

##### 3.1.1.1 设计理念：极简计分

**核心理念：** 让球友在球房拿起手机就能用，不需要学习复杂规则。

| 原则 | 说明 |
|------|------|
| 替代计分牌 | 进球点一下，代替摆球记数 |
| 替代扑克牌 | 犯规点一下，代替翻牌 |
| 替代脑子 | 谁赢谁输，系统帮你算 |

**交互哲学：**

```
┌────────────────────────────────────────────────────────┐
│  极简交互设计                                           │
├────────────────────────────────────────────────────────┤
│                                                        │
│   【进球】→ 不需记录（1-8 号球打完自然换下家）          │
│   【场次胜 / 9 号进】→ 点对应玩家的卡片                 │
│   【犯规】→ 点"犯规"按钮                                │
│   【改名】→ 长按玩家卡片                                │
│   【暂停 / 结束】→ 顶部工具栏                           │
│                                                        │
│   操作只集中在三件事：点卡片、点犯规、点顶栏。            │
│                                                        │
└────────────────────────────────────────────────────────┘
```

##### 3.1.1.2 中式八球计分

**设计原则：只记输赢，不记过程**

| 功能点 | 描述 | 操作 |
|--------|------|------|
| 人数 | 固定 2 人 | 一期只支持 2 人对局 |
| 抢几局 | 3 / 5 / 7 / 9 预设，外加自定义（≥1 任意整数） | 配置页选择 |
| 玩家名称 | 1 号位默认带入"我的昵称"，2 号位默认"对手" | 配置页输入框编辑；游戏中长按卡片也可改 |
| 本局胜 | 点击对应玩家卡片 → 确认 → 该玩家胜局 +1 | 点卡片 |
| 达成目标 | 胜局数达到抢几局目标时 toast 提示 | 自动 |
| 结束比赛 | 顶部工具栏 ✕ → 确认 → 保存记录 → 回首页 | 点顶栏 |

一期中八**不设单独的"犯规"按钮**，因为只记胜负，犯规既不加分也不扣分。规则判罚（8 号提前进袋、同桌进 8+球等）由玩家线下协商后再点"胜利者"即可。

**中八计分流程（极简版）：**

| 场景 | 操作 | 结果 |
|------|------|------|
| 某人赢本局 | 点击该玩家卡片 → 确认 | 该玩家 wins +1；未达目标继续，达到目标 toast |
| 改名 | 长按玩家卡片 | 弹出输入框改名 |
| 暂停 / 继续 | 顶栏 ⏸ / ▶ | 计时暂停 / 恢复 |
| 结束比赛 | 顶栏 ✕ → 确认 | 保存 MatchRecord → 回首页 |

##### 3.1.1.3 九球追分计分

**设计原则：以得分论英雄，记录每一杆的价值**

九球追分赛与中八不同，**不以局数论胜负，而是以累计得分定输赢**。

##### 赛前规则配置

比赛开始前，用户可配置比赛规则（目前的实装版本）：

| 配置项 | 默认值 | 可调整范围 | 位置 |
|--------|--------|------------|------|
| 比赛人数 | 3 人 | 2 人 / 3 人 | 主区 |
| 玩家名称 | 1 号位 = 用户昵称；2、3 号位 = 玩家2 / 玩家3 | 任意字符串（游戏中长按卡片也可改） | 主区 |
| 大金得分 | 10 分 | 正整数（默认推荐 8-15） | "得分规则"折叠区 |
| 小金得分 | 7 分 | 正整数（默认推荐 5-10） | "得分规则"折叠区 |
| 黄金9 得分 | 4 分 | 正整数（默认推荐 3-8） | "得分规则"折叠区 |
| 普胜得分 | 4 分 | 正整数（默认推荐 2-6） | "得分规则"折叠区 |

**一期明确不做**的配置项（避免首屏冗长 & 降低决策成本）：

- **目标分数**：九球追分改为**开放式累计**，不设胜负终点，由玩家自行约定何时"结束比赛"（点顶栏 ✕）。历史记录里会沉淀最终分差。
- **犯规补偿开关**：默认犯规者 -1 分，被补偿方 +1 分（见下节，幅度由 `foulCompensation` 决定，默认 1），不做"开/关"切换。
- **胜方开球**：球桌端自行管理，App 不跟踪"谁开球"。

##### 三人追分核心规则（球房常见玩法）

**座位顺序：** 1号位 → 2号位 → 3号位 → 1号位（逆时针轮换）

**追分逻辑：下家追上家**

| 情况 | 规则 | 示例 |
|------|------|------|
| 普胜 | 下家赢了上家的分 | 2号位赢了 → 2号位+分，1号位-分 |
| 大金/黄金9 | 赢所有其他人的分 | 2号位大金 → 2号位+分×2，1号位-分，3号位-分 |
| 犯规 | 自由球给下家，犯规者 -1，被补偿方 +1（任选） | 2号位犯规 → 3号位自由球，2号位 -1、可选给 3 号位/1 号位 +1 |

**得分计算示例（三人追分）：**

```
┌─────────────────────────────────────────────────────────────────┐
│                    三人追分得分规则                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   座位顺序：1号位 ← 2号位 ← 3号位                                │
│             上家   下家     下家                                   │
│                                                                  │
│   【普胜】2号位赢了1号位                                          │
│   → 2号位 +4分，1号位 -4分                                       │
│                                                                  │
│   【大金】2号位开球清台                                          │
│   → 2号位 +20分（10×2人），1号位 -10分，3号位 -10分             │
│                                                                  │
│   【犯规】2号位犯规                                               │
│   → 3号位自由球，2号位 -1 分；可选给 3号位或 1号位 +1 分         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**犯规规则说明：**

**犯规规则说明：**

| 犯规情况 | 自由球归属 | 分数归属 |
|----------|------------|----------|
| 犯规发生 | 下家获得自由球 | 犯规者 -1，被补偿方 +1（可选任意人） |

**进球得分规则（默认配置）：**

| 得分方式 | 分数 | 触发条件 | 得分对象 |
|----------|------|----------|----------|
| 大金 | 10分 | 开球方完成开球清台 | 赢所有其他玩家（自动计算） |
| 小金 | 7分 | 开球进球后传进9号/清台/开球直接进9 | 掏上家的分（需选择） |
| 黄金9 | 4分 | 开球直接打进9号球 | 赢所有其他玩家（自动计算） |
| 普胜 | 4分 | 普通胜利 | 掏上家的分（需选择） |
| 犯规 | ±1分 | 犯规时 | 犯规者 -1，可选任意人 +1 |

**进球判定说明（实装版）：**

玩家卡片是主要操作点位：**点击** → ActionSheet 选胜利类型 → 两步选择（普胜/小金）或直接应用（大金/黄金9）。

| 触发方式 | 胜利类型 | 后续步骤 | 得分结算 |
|----------|----------|----------|----------|
| 点玩家卡片 → 选"普胜" | 普胜 | 再选一次"掏谁的分"（2 人局自动选对方） | 赢家 +4，被掏者 -4 |
| 点玩家卡片 → 选"小金" | 小金 | 同上 | 赢家 +7，被掏者 -7 |
| 点玩家卡片 → 选"大金" | 大金 | 无 | 赢家 +10 × (人数-1)，其他各 -10 |
| 点玩家卡片 → 选"黄金9" | 黄金9 | 无 | 赢家 +4 × (人数-1)，其他各 -4 |
| 点"犯规"按钮 | 犯规 | ①选犯规者 ②选给谁 +1 分 | 犯规者 -1，获得者 +1 |

开球不进行追踪（不记为得分事件）。

##### 3.1.1.4 通用功能（顶部工具栏）

九球和中八进入游戏后，页面顶部常驻一个 `GameToolbar`：

| 区块 | 显示 / 操作 |
|------|-------------|
| 左 | 比赛标题（"九球追分" / "中式八球"） |
| 中 | ⏱ 计时器（`mm:ss`，超过 1 小时自动变 `HH:mm:ss`；暂停时灰色） |
| 右 | ⏸ 暂停 / ▶ 继续（切换）｜ ✕ 结束（二次确认） |

**结束比赛流程：**

1. 点 ✕ → 弹出"确认结束本场比赛？"
2. 确认 → 构造 `MatchRecord`：类型、结束时间戳、总耗时（含暂停扣除）、玩家快照（分数 / 胜局 / 统计）、冠军名
3. 记录写入本地持久化（`billiards-matches`）
4. 清空当前游戏 store，切回首页

**一期不包含**：分享战绩、斯诺克跳过、放弃比赛（用"结束"替代）。

#### 3.1.2 数据存储（本地缓存）

一期全部通过 Taro 存储适配层（H5 下是 localStorage，小程序下是 `wx.setStorageSync` 等）持久化：

| key | 内容 | 说明 |
|-----|------|------|
| `billiards-user` | `{ nickname, avatar }` | 用户昵称 + emoji 头像 |
| `billiards-matches` | `MatchRecord[]` | 每场比赛的快照，按 `endedAt` 倒序 |

`MatchRecord` 字段：`id / type / endedAt / durationMs / players[] / winnerName`。`players[i]` 含 `name, position?, score?, wins?, stats?`。

一期**不做**：IndexedDB、草稿比赛（未完成场次不保存，App 一刷新就没了——故意保持简单）、离线同步。

#### 3.1.3 用户基础功能

一期全部无登录（游客模式），通过"我"页面管理本机用户信息和战绩。

| 功能 | 实现 |
|------|------|
| 个人中心（"我"） | tab 栏第二个入口，包含昵称、头像、历史记录、关于 |
| 昵称编辑 | 点昵称行 → 自定义输入弹窗（H5/小程序通用） |
| 头像编辑 | 点头像 → 4×3 emoji 网格选择弹窗 |
| 历史记录 | 与"我"同页，按"今天 / 昨天 / YYYY/M/D"分组；每条显示：比赛类型、玩家、比分、时长、冠军、结束时间 |
| 删除记录 | 长按单条 → 确认弹窗 → 删除 |

一期**不做**：手机号/微信登录、头像上传（用 emoji 占位）、战绩分享。

## 4. 客户端规划

## 7. 竞品分析

### 7.1 现有竞品

| 产品 | 优点 | 不足 |
|------|------|------|
| 球房818 | 球房功能完善 | 计分功能简单，无追分赛 |
| 台球计分器 | 界面简洁 | 仅支持简单计分，无社交 |
| 约球神器 | 约球功能强 | 计分能力弱 |

### 7.2 差异化机会

1. **追分赛专属支持** - 目前无产品专门优化追分赛体验
2. **专业级计分** - 精准记录每一杆，符合竞技玩家需求
3. **轻社交** - 打球场景下的弱社交，不打扰核心体验
4. **球房赋能** - 帮助球房数字化运营

## 10. 附录

### 10.1 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 贴球 | Frozen Ball | 两球紧密接触 |
| 自由球 | Free Ball | 犯规后对手获得击球权 |
| 清台 | Clear Table | 打完己方所有目标球 |
| 飞球 | Jump Shot | 球跳起越过障碍球 |
| 连击 | Double Hit | 球杆碰触主球两次 |
| 普胜 | Normal Win | 九球追分赛中，普通单局胜利（清完部分球型后打进9号球），得4分，赢上家 |
| 小金 | Small Jack | 九球追分赛中，开球进球后传进9号球/清台(无进球)/开球直接进9号球，得7分，掏上家的分 |
| 大金 | Big Jack | 九球追分赛中，开球方完成开球清台（打进1-9号全部球），得10分，赢所有玩家 |
| 黄金9 | Golden 9 | 九球追分赛中，开球直接打进9号球，得4分，赢所有玩家 |
| 犯规 | Foul | 犯规时，下家获得自由球，犯规者可选择任意人+1分 |
| 胜方开球 | Winner Breaks | 九球追分赛规则，每局由获胜方开球 |
| 轮换 | Rotate | 多人中按顺序轮流击打 |
