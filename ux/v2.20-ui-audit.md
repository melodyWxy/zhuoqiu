# v2.20 UI 审计与四把刀

> 适用版本：v2.20（当前线上）
> 作用：在不动主色调（深绿 + 金）的前提下，按四个方向打磨 UI 的一致性、可读性与展示态。
> 关联：[`changelog/2026-05-28-01-ui-contrast-polish.md`] [`changelog/2026-05-29-04-weapp-bg-restore.md`]

## 背景

- v2.10 → v2.20 期间多次小步迭代，CSS 变量入口从 `:root` 改成 `page` 再合并成 `body + page`，留下不少历史样式；
- ui-contrast-polish 把卡片色板分了三档，但只覆盖了首页 / 我页 / 对局页，其他页面仍有硬编码色；
- 新增的省市区筛选 / 球房列表 / 反馈弹窗等，loading / 空态都还是兜底文字，没有完整展示态；
- 用户在迭代中沉淀了几条强 feedback：
  - **不改主色调**（金 + 深绿）
  - **改需求先改 PRD/设计稿再改代码**
  - **weapp 输入框调高度用 `height`，不堆竖向 padding**

## 不在范围内

- 主色调 / 设计风格的大幅改造（用户已明确否决）
- 业务功能新增（PRD 未涉及的 tab、页面、字段）
- 服务端 / 数据模型 / 网络层
- admin-web

## 四把刀

### 刀 A · 输入框高度 + 触达统一

**问题**：
- `InputModal` `LoginSheet` 输入框 `height: 48px`，按 weapp 真机 750rpx ≈ 96rpx，介于「能点」与「舒适」之间；
- `vp-region-cell`（球房列表省市区筛选）`padding: 8px 10px; font-size: 13px;` 实测高度仅 ≈32px，明显偏矮；
- `vp-clear-btn` `font-size: 12px; padding: 6px 8px;` 偏小；
- `me-more-btn` `ph-back` 按钮等已经在 32×32px 这一档，本身能用但触达边界。

**改法**：
1. 输入框统一 `height: 56px`（用 `height`，不动 `padding`，遵循 [[feedback-weapp-input-height]]）：
   - `InputModal .input-modal-field`
   - `LoginSheet` 输入框
   - `config` / `venue-apply` / `venue-login` 内的 input
   - `vp-search-input` 改成 `height: 44px`（搜索框单行可稍矮，但仍要 ≥44px）
2. 省市区筛选格 `vp-region-cell` `padding: 12px 12px; font-size: 14px;` →目测高度 ≈44px，且字号同 search-input；
3. 主操作按钮（确认 / 提交 / 搜索）触达 ≥ 44px 高度；
4. 图标小按钮（`ph-back` / `me-more-btn` / `vp-clear-btn`）确保 padding 后点击区 ≥ 32×32，关键操作再往上提到 ≥ 40×40。

**不做**：
- 不改字号系统（留给刀 C）；
- 不动 `:active` 反馈。

**验证**：
- weapp 真机：输入框点位舒适、不挡键盘；
- 省市区三个 cell 在窄屏 iPhone SE (375pt) 不挤、不溢出；
- H5 桌面浏览器无回退。

### 刀 B · 硬编码颜色 → CSS 变量化

**问题**（grep 已定位）：
- `legal/index.scss` 5 处硬编码：`#0a0f0d` `#e6ece8` `#d4af37` `#a0a8a4` `#d2d8d4`
- `InputModal` `LoginSheet` `MatchHistorySheet` `AvatarPickerModal` `FeedbackModal` 弹窗组件全用硬编码（`#1a2f23` `#fff` `#d4af37` `#0a0f0d` 等）
- `index/index.scss` `.join-card .mini-title { color: #93c5fd }`、`.observe-btn { background: #3b82f6 }`
- `tournament-detail/index.scss` `border-color: #60a5fa`
- `config/index.scss` `background: #fff`（可能是 switch 圆球，需保留）
- `nine-ball/index.scss` `linear-gradient(135deg, #f5d76e 0%, ...)` 金色亮色硬编码
- `--bg-page` 在 8 处文件被引用为 `var(--bg-page, #0d1013)`，但 global.scss 从未定义过 `--bg-page`，永远走 fallback
- `app.config.ts` tabBar 颜色 `#a0a8a4` `#d4af37` `#1a2f23` 与 token 没有显式映射

**改法**：
1. `global.scss` 的 `@mixin color-tokens` 内补充：
   - `--bg-page: #0d1013;`（与现有 fallback 一致，正式落到 token）
   - `--accent-bright: #f5d76e;`（九球胜分用的亮金）
   - `--info-deep: #3b82f6;`（联机观众蓝按钮）
   - `--info-light: #93c5fd;`（联机标题浅蓝）
2. 把 `legal/index.scss` 全部硬编码替换为 `var(--bg-dark)` / `var(--text-primary)` / `var(--accent)` / `var(--text-secondary)`，正文换 `var(--text-primary)` 调一档（保留视觉感）
3. 弹窗组件统一用变量：`#1a2f23` → `var(--primary)`，`#fff` → `var(--text-primary)`，`#a0a8a4` → `var(--text-secondary)`，`#d4af37` → `var(--accent)`，`#0a0f0d` → `var(--primary-dark)`
4. `app.config.ts` 在 ts 顶部加注释明示色值与 token 的对应（weapp 不能用 var，只能注释级同步）
5. `--bg-page` fallback 去除（token 落地后不需要）

**不做**：
- 不动 slot 渐变里的次色（`#15242e` `#261a2c` 等是为了渐变深尾，token 化收益低）；
- 不动 `ConnectionBanner` 的状态色（warning/error 已经用 token，但部分还在硬编码——刀 B 范围内顺手改了即可，不单列）。

**验证**：
- 视觉零回归（硬编码值与 token 值一致，纯重命名）；
- legal 页面 H5 + weapp 截图前后一致；
- 弹窗（输入 / 登录 / 头像 / 反馈 / 历史）截图前后一致。

### 刀 C · 状态 / 层级一致性 token

**问题**：
- disabled opacity 三档（0.4 / 0.5 / 0.55 / 0.7）；
- 圆角 `8px` `10px` `12px` `14px` `16px` 五档混用；
- 字号无系统：11/12/13/14/15/16/17 px 全员上岗；
- 文本截断：单行 `nowrap+ellipsis`、双行 `-webkit-line-clamp: 2`、三行未限定，散落各处；
- modal 遮罩 `rgba(0,0,0,0.55) z-index: 9999` 在 5 个弹窗组件里独立定义。

**改法**（先加 token，渐进替换）：
1. `global.scss` 补充：
   ```scss
   --opacity-disabled: 0.4;
   --opacity-active: 0.85;
   --radius-sm: 8px;
   --radius-md: 12px;
   --radius-lg: 16px;
   --font-size-caption: 12px;
   --font-size-body: 14px;
   --font-size-title: 16px;
   --font-size-heading: 20px;
   ```
2. 加两个 SCSS mixin（与 `color-tokens` 同档）：
   ```scss
   @mixin text-truncate-1 { overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }
   @mixin text-truncate-2 { display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical; overflow: hidden; }
   @mixin modal-mask { position: fixed; inset: 0; background: rgba(0, 0, 0, 0.55); z-index: 9999; display: flex; align-items: center; justify-content: center; }
   ```
3. 把弹窗组件的遮罩改用 `@include modal-mask`；
4. 把每页的截断写法改用 mixin；
5. 字号 / 圆角 / opacity 渐进替换：本次只在弹窗 + 我页 + 球房列表三个文件内做替换，其余下一轮做（避免一次改太大）。

**不做**：
- 不动 padding/margin spacing 系统（spacing 比颜色更需要 case-by-case，留给后续）；
- 不强制所有页面都替换 token（先布点，避免一次性大改）。

**验证**：
- weapp + H5 视觉无回归；
- mixin 在 weapp 编译后 `dist/app.wxss` 展开正确（color-tokens 已有先例可参考）。

### 刀 D · 展示态优化（空态 / loading / 选择框）

**问题**（用户原话 + 排查所得）：
- 球房列表搜不到 → 现在仅 `<View>暂无球房</View>` 一行字
- 球房列表 loading → 仅 `<View>加载中…</View>` 一行字
- 省市区选择框过矮过紧（已在刀 A 同步改尺寸，刀 D 这里负责字号 + 视觉）
- 赛事 / 球房 / 历史等列表的空态、loading 是否一致也需要顺手扫盲
- 反馈弹窗 / 我页 / 比赛详情等异步页面没有统一的 loading 体感

**改法**：
1. **空态组件**：在 `components/` 下新建 `EmptyState/`（emoji + 标题 + 描述 + 可选 CTA）：
   ```
   🎱
   暂无球房
   换个城市试试，或下拉刷新
   ```
   - 球房列表 empty / 赛事列表 empty / 历史 empty 三处先接入；
   - 用 emoji 而非图片资源（与现有「emoji 作 icon」风格一致，也避免 weapp 静态资源拖大包体）。
2. **Loading 组件**：在 `components/` 下新建 `LoadingState/`（金色三点 / 球转动动画）：
   - weapp 兼容：用 CSS keyframes 在文本前加圆点动画；
   - 复用到球房 / 赛事 / 详情页 / 反馈提交。
3. **省市区选择框**：刀 A 已经把高度改到 ≈ 44px，刀 D 再加：
   - 字号 13 → 14
   - `vp-region-cell` 加一个右侧下拉箭头（`▾`）的明确视觉提示，不再藏在文字里
   - disabled 态视觉降级（更深、不可点）
4. **下拉刷新提示**：球房 / 赛事 / 我页比赛历史的 `usePullDownRefresh` 真机有「刷新中」原生气泡，但 H5 没有；H5 加 `pull-to-refresh` 简易兜底（可选，本轮先不做，记 TODO）。

**不做**：
- 不做骨架屏（weapp 包体限制，骨架屏 SCSS 写起来啰嗦，用 LoadingState 已足够）；
- 不做错误页（异常处理目前是 toast，已 OK）；
- 不接入第三方 UI 库（与 mixin / token 体系冲突）。

**验证**：
- 球房列表清空 keyword 切到一个无数据的省 → 看到空态；
- 球房列表初始加载 → 看到 LoadingState 而非「加载中…」；
- 省市区切换 → 高度、字号、disabled 态都正确；
- 真机 + H5 双端回归。

## 落地节奏

每把刀独立 commit + 独立 changelog：

| 顺序 | 内容 | 改动面 | 风险 |
|------|------|--------|------|
| 1 | 刀 B（颜色 token 化）| global.scss + 弹窗 + legal + 散点 | 低（语义重命名）|
| 2 | 刀 A（输入 + 触达）| 输入 / 选择 / 按钮 | 中（数值改动可能挤布局，需真机/H5 双端对照）|
| 3 | 刀 C（状态 / 层级 token）| global.scss + 弹窗 + 我页 + 球房 | 中（mixin 替换，渐进）|
| 4 | 刀 D（展示态）| 新增 EmptyState/LoadingState 组件 + 接入 | 中偏高（新组件 + 接入面广）|

> 顺序考虑：先 B（纯重命名零回归）让后面三刀有 token 可用；再 A（数值微调）；再 C（落 token）；最后 D（新组件）。每刀完后真机 + H5 截图回归一次。
