> 配套 PRD：`prd/billiards-match-app-prd-v2.10.md` §3

> 注：C 端入驻入口单独见 `03-c-side-venue-and-tournament.md` §7.5-§7.7。本文档专注 admin 域名下的流程。

## 1. 入口与导航

```
URL                         谁能访问                   目的
─────────────────────────────────────────────────────────────────
admin.zhuoqiu.xxx/apply     匿名可见                   入驻引导页
admin.zhuoqiu.xxx/login     匿名可见                   登录（手机号+验证码）
admin.zhuoqiu.xxx/          登录后                     仪表盘（按角色分支）
admin.zhuoqiu.xxx/venue/... 已开店的商家               店铺管理
admin.zhuoqiu.xxx/admin/... 平台运营                   平台后台（已有）
```

登录后的侧边栏按账户类型出菜单：

| 账户类型 | 菜单 |
|---------|------|
| `platform_admin` | 概览 / 用户 / 比赛 / **入驻审核** / **赛事总览** / 审计 |
| `venue_owner` | 店铺主页 / 店铺资料 / 赛事管理 / 报名列表 / 现场控台 / 团队（占位） |
| `venue_staff`（v2.11） | 赛事管理 / 报名列表 / 现场控台 |
| 未入驻商家 | 申请入驻（只此一个菜单） |

交互要点：
- 每步填完自动保存到 `venue_application.payload`（jsonb），刷新不丢
- 图片上传走 OSS 直传：前端先向后端拿 policy（STS token）→ 直传 → 拿回 URL → 回传给后端写入 application
- "提交审核" 校验所有必填字段，成功后切换到状态页 `/apply/status`

通过后：
- 创建 `venues` 记录，owner 绑定提交人账号
- 应用的 state = approved，venue_id 回填
- 发短信 + 站内信"恭喜通过"
- 商家下次登录直接进店铺主页

驳回：
- state = rejected，reject_reason 存下
- 发短信 + 站内信
- 商家可修改后重新提交（state 回到 draft）
