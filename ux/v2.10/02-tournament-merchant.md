## 1. 菜单与状态

商家侧的赛事一级菜单：

```
我的球房
├─ 概览（已在 01-venue-onboarding §5）
├─ 赛事管理 ← 本文档
│   ├─ 赛事列表
│   ├─ 新建 / 编辑赛事
│   ├─ 报名列表
│   └─ 现场控台（bracket 推进）
└─ 店铺资料
```

赛事状态机：

```
draft ──publish──▶ published ──(到达 reg_starts_at)──▶ registering
                                 │
                                 │ 手动/到达 reg_ends_at
                                 ▼
                              registration_closed
                                 │
                                 │ 手动"开赛"
                                 ▼
                             in_progress ──(冠军决出)──▶ completed
                                 │
                                 └── cancelled（任何阶段可手动取消）
```

```
│                                                                │
```


**每个对阵卡的行为**（取决于 `bracket_match.state`）：
- `pending`：一方或双方未定（上一轮未结束）→ 灰色不可点
- `ready`：双方都到位 → 可点 "开始比赛" 生成 match → 跳现场控台弹窗 / 新标签页
- `in_progress`：关联 match 进行中 → 蓝色脉动 → 点"查看实时记分"
- `completed`：有 winner → 绿色 → 点"查看结果"

## 5. 页面：现场控台 / 实时记分对接（/venue/tournaments/:id/live）

赛事开打后，商家的"大屏模式"：

```
┌────────────────────────────────────────────────────────────────┐
│ 🏆 五一擂台赛  进行中                    回合 R2/R4            │
├────────────────────────────────────────────────────────────────┤
│ 正在进行（2 场）                                               │
│  ┌──────────────────────┐  ┌──────────────────────┐            │
│  │ 张小球  12 :  8  李快手│  │陈稳准   5 : 11  刘快枪│            │
│  │ [开大屏]  [查看详情]  │  │[开大屏]  [查看详情] │            │
│  └──────────────────────┘  └──────────────────────┘            │
│                                                                │
│ 待开赛（2 场）                                                 │
│  ┌──────────────────────┐  ┌──────────────────────┐            │
│  │ 王稳定  vs  赵三炮    │  │孙老六   vs  何瞄准   │            │
│  │ [ 开始比赛 ]          │  │[ 开始比赛 ]         │            │
│  └──────────────────────┘  └──────────────────────┘            │
│                                                                │
│ 已结束（4 场）                                                 │
│  ...                                                           │
└────────────────────────────────────────────────────────────────┘
```

"开始比赛"的行为：
1. 后端创建 `matches` 一条记录：
   - 类型 = 赛事项目
   - rules = 赛事 rules
   - 两个 slot 自动占位为 bracket 两方的 display_name + user_id
   - venue_id = 赛事的 venue_id
   - tournament_bracket_match_id = 该对阵 id
2. bracket.state = in_progress
3. 前端 redirect 到该 match 的商家"大屏"页（新样式，字更大，便于桌边观看）
4. match 走正常的 event 流程
5. match.end：
   - winner = 赢家 slot 的 user_id
   - 服务端 hook 自动回填 bracket.winner_registration_id
   - 若本轮最后一场完成 → 生成下一轮对阵
   - WS 推 `tournament:{id}` 事件 → 赛程页自动刷新

## 7. 完赛

最后一场 match.end → bracket 最后一条 completed → 赛事 state=completed。

页面上：
- 赛程树顶端出"冠军：张小球"
- 报名列表多个"名次"列（由 bracket 结果回算）
- 商家可点"导出战绩 CSV"

## 8. 边界 / 兜底

| 场景 | 处理 |
|------|------|
| 人数不到下限到时间 | 商家可延长报名时间 or 取消赛事 |
| 报名后弃权 | 报名列表 [取消] → state=withdrawn，bracket 里对手自动 walkover |
| 开赛后玩家没到 | 商家在控台点 "标记弃权" → bracket 直接推进 |
| 比赛记错 | 走比赛自身的 undo；若已 end 则商家申诉（v2.11+）|
| 奇数人数 | 自动补 BYE 到 2^n；头部种子享 BYE |
| 赛事进行中商家临时取消 | 允许，所有未开始对阵的 match 不创建；已完成的保留 |
