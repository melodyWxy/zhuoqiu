## 0. 两套视角一个客户端

同一个 Taro C 端，**根据当前登录态**呈现两种完全不同的 UI。

| 登录态 | 底部 Tab | 关键特征 |
|--------|----------|----------|
| 未登录 / C 端用户（现状） | 首页 / 球房 / 赛事 / 我的 | 可开局、可报名、可发现球房 |
| C 端用户 + 球房账号双登录 | 首页 / 球房 / 赛事 / 我的 | 同上 + "我的"页新增"球房管理模式"小条目 |
| 切入球房管理模式后 | 首页 / **我的球房** / **我的赛事** / 我的 | 只读视角；"创建"类动作引导去后台 |

切换机制：

```text
状态 1: 只有 C 端 user session → 正常视角
状态 2: 我的页 → 🏢 切换球房管理模式 → 输入商家手机号+验证码 → 建立 venue session
状态 3: 双 session 共存；UI 渲染优先看 venue session；我的页顶部出 [ 退出球房视角 ] 回状态 1 样的 UI
状态 4: 点"退出球房视角"不登出，只是切 UI；再切回去无需重新登录
```

存储：`auth-store.ts` 加 `venueSession: { accessToken, refreshToken, account }` 字段；加 `viewMode: 'user' | 'venue'` 渲染开关。

## 1. Tab 重构

v2 目前 Tab：`首页 / 九球 / 中八 / 我的`。

v2.10 新增两个实体入口，但保持 Tab 数 ≤ 5（小程序限制 + 体验）：

**方案 A：把"九球/中八"收进首页（推荐）**

```
底部 Tab: [ 首页 ]  [ 球房 ]  [ 赛事 ]  [ 我的 ]
```

- 首页承担"开一局比赛"的入口（九球 / 中八 / 加入联机）
- 球房 = 发现店铺、去店里报名
- 赛事 = 赛事广场 + 我的赛事
- 我的 = 资料、历史、设置

**方案 B（保留原 Tab，新入口放首页）** ——备选，如果不想改导航：

```
底部 Tab: [ 首页 ]  [ 九球 ]  [ 中八 ]  [ 我的 ]
首页顶部加横幅：「🏆 赛事广场」「🏢 附近球房」两个卡片跳转
```

**默认采用 A**。
