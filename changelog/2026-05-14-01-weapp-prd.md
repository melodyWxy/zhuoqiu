# 微信小程序适配 PRD（4 个共性问题方案）

## 动机

用户在 zhuoqiu C 端跑微信小程序时，发现 4 个共性问题，需要先把方案落到 PRD 再改代码（项目规矩：PRD 先行）：

1. tabBar 图标用 svg，weapp 编译后空白
2. `global.scss` 用 `:root { … }` 声明 CSS 变量，weapp 不识别 → 全局深色底丢失
3. weapp / h5 偶发"整页被缩放"（字号问题）
4. `npm run build:weapp` 没注入域名，请求落到 `localhost:3001`

## PRD / 设计变化

- 新增：`prd/weapp-adaptation.md` —— 「击球帮 · 微信小程序适配技术方案」
  - 0. TL;DR：4 项问题对照表
  - 1. SVG → PNG（tabBar 4 张图，81×81~162×162 PNG-32）
  - 2. `:root` → `page`（合并到同一个 `page` 选择器）
  - 3. 字号治理（page 兜底 `font-size: 32rpx` + `text-size-adjust: 100%`，业务保持 px 由 pxtransform 处理；禁用 `html{font-size}` / `body{zoom}`）
  - 4. `build:weapp:prod` 脚本 + cross-env 注入 `TARO_APP_API_BASE` / `TARO_APP_WS_BASE`
  - 5. 落地顺序与 changelog 编号
- 用户确认时间：2026-05-14（用户原话："按你的思路继续推进"）
