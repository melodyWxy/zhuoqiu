# MVP 落地：首页/九球/中八/我 四大模块实装

> **说明**：本条日志合并本次会话的多轮迭代，用于回溯"从脚手架到可用"这一阶段的决策。之后每轮改动会按一事一记拆开。

## 动机
初版代码与 PRD 脱节严重：
- 首页 3 个卡片直接跳游戏页，赛前配置页是孤儿
- 玩家名字无法修改（写死 张三/李四/王五）
- 九球"普胜 / 犯规"按钮自动扣上家分，与 PRD 两步选择规则不符
- 还有"换手"按钮，和极简理念冲突
- 没有个人中心；历史记录是 mock 数据，结束比赛也不会保存
- 无计时能力，无暂停/结束比赛操作
- 九球 得分规则写死，不可配置
- H5 平台 `Taro.showModal editable` 与长 `showActionSheet` 不实现，导致改名/头像交互崩溃
- tab 栏缺图标

## PRD / 设计变化
由于本条是回溯性记录（改动已经发生），PRD 与设计稿与代码在本次一并对齐到 v1.7.0（见两文件的修订记录）。主要变更点：
- **首页**：3 卡 → 2 卡（九球、中八）；卡片不再直达游戏页，先进 config
- **Tab 栏**：首页 / 记录 / 我 → 首页 / 我（历史记录并入"我"）
- **赛前配置**：
  - 九球移除"目标分数"（改为开放累计）
  - 九球得分规则 大金/小金/黄金9/普胜 可自定义（折叠"高级"区）
  - 中八新增自定义抢局数（默认 5，预设 3/5/7/9 + 自定义）
  - 1 号位默认带入"我的昵称"
- **九球计分页**：
  - 移除 普胜/小金/大金/换手 直通按钮；只保留"犯规"
  - **点击玩家卡片** → ActionSheet 选 普胜/小金/大金/黄金9 → 普胜/小金 再选掏谁的分；大金/黄金9 自动
  - 长按玩家卡片 → 改名（自定义 Modal，H5 可用）
  - 犯规按钮 → 两步选择（选犯规者 → 选给谁+1）
  - 玩家卡片统计多加 👑（黄金9 计数）
  - 顶部工具栏：标题 + ⏱计时 + ⏸暂停/▶继续 + ✕结束
- **中八计分页**：
  - 移除"换手"按钮
  - **点击玩家卡片** → 确认本局胜 → +1 胜；达目标 toast
  - 长按改名
  - 顶部同样的 GameToolbar
- **我的页面**：昵称（点击改）+ 头像（emoji 网格选）+ 历史记录（按日期分组，长按删除）
- **结束比赛**：
  - 弹窗确认 → 构造 `MatchRecord`（type / 时长 / 玩家分数或胜局 / 冠军）→ 保存到 matches store → 清空 game state → 切回首页
  - 记录持久化到 LocalStorage（Taro 存储适配 H5 + 小程序）

## 遗留问题 / 已知限制
- Tab icon 使用 SVG，在微信小程序里可能需要替换为 PNG（H5 OK）
- 历史记录目前只读 + 单条删除；无多选 / 批量清空
- 计时器在页面跳转场景下（非正常退出、刷新）会丢状态——未做崩溃恢复
- 中八没有"犯规"按钮（按 PRD 中八只记胜负，犯规不单独记录）
- 二期功能（云同步、球友圈、球房）未动
