# 二期批次 1 · 共享比赛端到端打通

## 动机

上一轮完成了服务端脚手架 + 管理端读接口。本轮补齐共享比赛的**核心业务 API** —— 从用户登录到创建房间、扫码加入、记分、撤销、结束 → 管理员介入（强制结束/封禁）+ 审计日志 → 全链路通。

## PRD / 设计变化

无新增。完全按 `ux/v2/shared-match-backend.md` 已有设计实现。

## 代码变化

### 新增文件

- **用户模块**
  - `src/users/users.service.ts`（+ module）：upsertByPhone / upsertByWechat / upsertByDouyin（含 unionId 跨小程序合并）+ updateProfile
- **C 端 Auth**
  - `src/auth/user-auth.controller.ts`：`/v1/auth/{wechat,douyin,phone/send-sms,phone/verify,refresh,logout}`
  - `src/auth/user-auth.guard.ts` + `current-user.decorator.ts`
  - `src/auth/sms.service.ts`：MVP 短信（打日志，不接真实通道）
  - `src/auth/dto/client-login.dto.ts`
  - `src/auth/jwt-payload.ts`：新增 UserRefreshPayload
  - `src/auth/auth.service.ts`：扩展 issueUserTokens / refreshUserAccessToken / verifyUserAccessToken
- **Me**
  - `src/me/me.controller.ts`（+ module）：`GET /me` / `PATCH /me`
- **Match 状态机（纯函数）**
  - `src/match/state-machine/types.ts`：MatchEventPayload 联合类型
  - `src/match/state-machine/nine-ball.ts`：applyNineBallEvent + validateNineBallEvent（含 score_normal_win / small_jack / big_jack / golden9 / foul）
  - `src/match/state-machine/eight-ball.ts`：applyEightBallEvent + validate
- **Match API**
  - `src/match/match.service.ts`：create / detail / joinByCode / occupySlot / leaveSlot / appendEvent / undoLast / endByOwner / forceEndByAdmin / forcePauseByAdmin / kickByAdmin / listMyMatches
    - 用 **PG `pg_advisory_xact_lock`** 做房间级串行锁，保证事件 serverSeq 单调、状态一致
    - 每次 GET 详情**从事件重算 scores/stats**，自然支持撤销
  - `src/match/match.controller.ts`：`POST /matches`、`/matches/join`、`GET /matches/:id`、`/matches/:id/{seat,events,events/undo,end}`、`GET /me/matches`
- **审计 & 管理写**
  - `src/audit/audit.service.ts`（+ module，Global）：写 admin_audit_logs
  - `src/admin/matches-admin-write.controller.ts`：force-pause / force-end / kick
  - `src/admin/users-admin-write.controller.ts`：ban / unban
  - `src/admin/audit-admin.controller.ts`：`GET /admin/audit-logs`（operator/readonly 只能看自己）
  - `src/admin/dto/admin-write.dto.ts`
- **工具**
  - `src/common/utils/id.ts`：genId / genMatchCode（6 位大写，排除易混字符）
- **main.ts**：全局 `BigInt.prototype.toJSON = Number` 解决 Prisma BigInt 序列化

### 修复 bug

- `occupySlot`：原本判到"occupant.userId !== userId"就报已占，但空 userId（null）被错误拒绝；加 `!== null` 前置判空
- `UsersService.isBanned`：返回类型 `boolean | Promise<boolean>` 不合法，改为 `Promise<boolean>`

## 遗留问题 / 已知限制

- **WebSocket 未实现**：C 端目前靠 POST 同步返回拿新状态；多端实时同步要等 Realtime Hub
- **账号管理 CRUD 未实现**（创建/编辑/重置密码 admin_accounts）
- **用户合并 / 手机号绑定 / 解绑**接口未实现（`/me/bind-phone`、`/admin/users/merge` 等）
- **微信 / 抖音登录是 mock**：没真正换 openId；接入生产需要 appId/appSecret
- **短信通道是 mock**：验证码直接打日志
- **Refresh token rotation 未做**：refresh token 可重复使用
- **Analytics 指标只有 overview**，详细图表指标未做
- **无 Redis**：单实例够用；多实例部署再加
- **无 Docker / CI**
