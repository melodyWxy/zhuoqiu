# v2.10 P1：入驻申请 + 平台审核 + 文件上传 + 三端接入

承接 P0 的商家账号登录，本期把完整的入驻闭环跑通：

```
商家填表 + 上传营业执照 → 提交申请 → 平台审核
 ├─ 通过 → 自动建 Venue + 绑定 owner + 审计日志
 └─ 驳回 → 带原因，商家可修改后重提
```

## 服务端

### `src/venue/` 扩展

- `venue-application.service.ts`：
  - `submit`：同一账号的 draft/pending/rejected 申请会被覆盖（不新增记录），保证一人一个 active 申请
  - `listForAdmin` / `getByIdForAdmin` / `approve` / `reject`
  - `approve` 走事务：建 Venue → 绑定 owner → 更新 application → 写 AdminAuditLog
- `venue-application.controller.ts`：`POST /v1/venue/applications`、`GET /mine`（带 VenueAuthGuard，双 client 都可用）
- `venue-admin.controller.ts`：`GET/POST /v1/admin/venue-applications/*`（带 AdminAuthGuard + `@Roles`）
- DTO 校验：ApplicationPayloadDto（name 2-128 / phone 8-15 / tablesCount 1-200 / openHours 7 项）

### `src/upload/` 新模块（MVP 本地存储）

- `POST /v1/uploads?category=license`，multer 限制 5MB + 图片 MIME
- `main.ts` 挂 `app.useStaticAssets(uploadRoot, { prefix: '/uploads/' })`
- 受 VenueAuthGuard 保护（登录后可用）；生产上换 OSS 接口 schema 保持兼容

### 依赖

- 新增：`multer` + `@types/multer`

## Admin-web 平台侧

- `pages/Venues/Applications.tsx`：列表（状态筛选 + 来源标签 + 分页）
- `pages/Venues/ApplicationDetail.tsx`：详情 + 营业执照预览 + 通过/驳回弹窗（驳回带原因）
- `routes.tsx` 注册 `/venue-applications` 和 `:id`
- `AppLayout.tsx` 侧边栏加"球房入驻审核"菜单（ShopOutlined）
- `api/venues.ts`：admin 侧用 admin-web 的 http 实例

## Admin-web 商家侧

- `stores/venue-auth.ts`：独立 zustand store（与 admin token 分离）
- `api/venue-client.ts`：独立 axios 实例，request 带 venue token，401 自动 refresh
- `api/venue.ts`：venueAuthApi / venueApplicationApi / uploadApi
- `pages/VenueMerchant/VenueLogin.tsx`：`/venue-login`，手机号+验证码登录（硬编码 client=admin_web）
- `pages/VenueMerchant/Apply.tsx`：`/apply`，分段表单 + 营业执照直传
- `pages/VenueMerchant/ApplyStatus.tsx`：`/apply/status`，状态态（pending / rejected / approved），已绑定 venue 时展示店铺摘要
- 登录后自动跳：有 venue → status；无 venue 且无 pending → apply；有 pending → status

## C 端 Taro

- `core/auth/store.ts`：加 `venueSession` + `viewMode: 'user' | 'venue'`，与 C 端 user session 并存
- `core/api/venue-client.ts`：独立 callVenueApi，用 venueSession token，401 走 `/venue-auth/refresh`
- `core/api/venue.ts`：venueAuthApi / venueApplicationApi + `uploadVenueFile`（基于 `Taro.uploadFile`，multipart 兼容）
- `pages/me/index.tsx` + `index.scss`：新增 `venue-mode-card`
  - 未登录商家 → "🏢 切换到球房管理模式"
  - 已登录商家 → "当前已登录商家 · 查看球房状态 →"
- 新页 `pages/venue-login/`：独立手机号+验证码登录，client=c_app
- 新页 `pages/venue-apply/`：表单 + 上传 + 三态展示（审核中 / 驳回 / 已绑定 venue）
- `app.config.ts` 注册两个新路由

## 验证（本地 curl + TS 编译）

```text
✓ venue-auth/sms/send + verify(admin_web) → access token
✓ POST /v1/uploads?category=license → 返回 URL，/uploads/ 静态可回读
✓ POST /v1/venue/applications → pending 记录落库
✓ admin login → GET /v1/admin/venue-applications?status=pending → 可见刚刚的申请
✓ POST /v1/admin/venue-applications/:id/approve → 返回 venue + application approved
✓ venue-auth/me → account.venueId / venue 非空
✓ 审计日志 admin_audit_logs 多一条 action=approve_venue_application
✓ server TS 编译通过；admin-web / Taro 均无新增错误
```
