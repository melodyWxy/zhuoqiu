# UI 审计·刀 B - 硬编码颜色 → CSS 变量化

## 动机
`ux/v2.20-ui-audit.md` 立的 4 把刀里的第 1 把（也是顺序上的第 1 把，因为是
纯语义重命名零回归）。背景：

- `legal/index.scss` 全部颜色硬编码（5 处），与全局色板脱节，将来要调主色
  必须改两套
- 5 个弹窗组件（`InputModal` `LoginSheet` `MatchHistorySheet`
  `AvatarPickerModal` `FeedbackModal`）大量直接写 `#1a2f23` `#fff` `#a0a8a4`
  `#d4af37` `#0a0f0d` 这一组色（其实就是 `--primary` `--text-primary`
  `--text-secondary` `--accent` `--primary-dark`），各写各的
- 散点页面里有 `.join-card .mini-title { color: #93c5fd }`、
  `.watch { background: #3b82f6 }`、`.win-golden9 { ... #f5d76e ... }`、
  `.td-bracket-in_progress { border-color: #60a5fa }`、`.hot-tag-registering
  { color: #60a5fa }` 等硬编码
- `--bg-page` 在 8 个文件里被引用为 `var(--bg-page, #0d1013)`，但
  `global.scss` 从未定义过 `--bg-page`，永远走 fallback；其他类似 fallback
  写法（`var(--error, #ef4444)` 等）也散落各处
- `app.config.ts` 顶栏 / tabBar 颜色与 token 没有显式映射关系（weapp 顶栏
  不能用 var，但起码注释级要同步）

刀 B 是「先把 token 入口补齐 + 把硬编码改用 var」，**不动任何视觉**，给后面
A/C/D 三刀提供 token 抓手。

## PRD / 设计变化
- `ux/v2.20-ui-audit.md` 里「刀 B」一节为本轮唯一参考；不涉及 PRD 字段改动。
- 设计 token 新增（写进 `styles/global.scss` `@mixin color-tokens`）：
  - `--bg-page: #0d1013;` —— 页面 wrapper 背景，token 化原来 8 处的 fallback
  - `--info-deep: #3b82f6;` —— 联机观众蓝按钮（饱和蓝）
  - `--info-light: #93c5fd;` —— 联机标题浅蓝
  - `--accent-bright: #f5d76e;` —— 九球金球渐变高光

## 代码变化

### 入口
- `billiards-score/src/styles/global.scss`：`@mixin color-tokens` 内补充 4
  个新变量（见上）
- `billiards-score/src/app.config.ts`：在 `window` / `tabBar` 上方加一段注释
  说明 hex 与 token 的对应关系（weapp app.json 不接 CSS 变量，只能字面量）

### 弹窗组件全部 token 化
- `components/InputModal/index.scss`
- `components/LoginSheet/index.scss`
- `components/MatchHistorySheet/index.scss`
- `components/AvatarPickerModal/index.scss`
- `components/FeedbackModal/index.scss`

替换规则：
- `#1a2f23` → `var(--primary)`
- `#fff` / `#ffffff` → `var(--text-primary)`
- `#a0a8a4` → `var(--text-secondary)`
- `#d4af37` → `var(--accent)`
- `#0a0f0d` → `var(--primary-dark)`
- `#ef4444` → `var(--error)`

### legal 页全替换
- `pages/legal/index.scss`：`#0a0f0d` `#e6ece8` `#d4af37` `#a0a8a4`
  `#d2d8d4` 全部改用变量；正文换 `var(--text-primary) + opacity: 0.92` 兼顾
  「比纯白略柔和」的视觉感。

### 散点页面
- `pages/index/index.scss`：`.join-card .mini-title #93c5fd` →
  `var(--info-light)`；`.hot-tag-registering #60a5fa` → `var(--info)`
- `pages/join/index.scss`：`.watch #3b82f6/#ffffff` → `var(--info-deep) /
  var(--text-primary)`
- `pages/tournament-detail/index.scss`：`.td-bracket-in_progress #60a5fa` →
  `var(--info)`
- `pages/nine-ball/index.scss`：`.win-golden9` 渐变 `#f5d76e` →
  `var(--accent-bright)`
- `pages/config/index.scss`：toggle knob `#fff` → `var(--text-primary)`

### Fallback 统一去除
8 处 `var(--bg-page, #0d1013)` → `var(--bg-page)`；3 处 `var(--error, #ef4444)`、
`var(--accent, #d4af37)`、`var(--primary-dark, #0a0f0d)` 同样去 fallback。

涉及文件：
- `pages/venue-apply/index.scss`
- `pages/tournament-detail/index.scss`
- `pages/venue-detail/index.scss`
- `pages/venues/index.scss`
- `pages/me/index.scss`
- `pages/tournaments/index.scss`
- `pages/venue-login/index.scss`
- `components/PageHeader/index.scss`

## 故意保留的硬编码
- **slot 渐变次色**（`#15242e` `#261a2c` `#1d2616` `#2e1d18` `#3a5a48`
  `#1f3a2c` `#3f5a3a` `#2a4530`）：渐变深尾，token 化收益低，刀 C 也不动。
- **`ConnectionBanner` 状态色变体**（`#d97706` `#f59e0b` `#15803d` `#ef4444`
  `#f87171`）：是为了在浅色 banner 背景上保持 AAA 对比度的 contrast-tuned
  variants（amber-700 / green-700 / red-400 等），与现有 `--warning`
  `--error` `--success` 不是 1:1 关系。强行替换会改视觉，留待后续若做
  `--warning-text` `--error-text` 这一档 token 时再统一。
- **`FeedbackModal .feedback-modal-type-chip color: #c8d0cc`**：介于
  `--text-secondary` 与 `--text-primary` 之间的 chip 默认字色。同样等后续
  `--text-secondary-strong` 类 token 时再统一。
