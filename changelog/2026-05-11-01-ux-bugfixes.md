# 联机模式 UX 修复 + 历史详情页 + 房间码横幅

## 动机

同事局域网测试反馈 9 个问题，全是体验阻断或易用性差：看不到房间码、白底白字按钮、结束后停留在游戏页、观众无法退出……集中一轮修完。

## 用户反馈原文（9 条）

1. 九球比赛页看不到房间号，无法分享给朋友
2. 管理后台也看不到那个房间号
3. "以观众身份进入"按钮白底白字，看不到
4. 加入联机必须登录（目前未检查）
5. 占位进入时要用自己昵称覆盖原昵称
6. 结束游戏后应跳到"我的"页（能看到历史）
7. 历史记录要能点进去看详情（比分、操作日志）
8. 登录用户默认联机，未登录默认本地；切换联机时要求登录
9. 联机房间的玩家/观众要能退出
10. 房主新开联机比赛时，自动关闭旧房
11. 关房时推送所有其他人，弹"比赛已结束，3s 自动退出"，跳我的

（原描述合并了几项，实际代码上拆成了 9 个 task）

## 代码变化

### C 端（Taro）

- `pages/nine-ball/OnlineMode.tsx` + `pages/eight-ball/OnlineMode.tsx`：
  - **房间码大横幅**（金色 32px 大字，点击复制）
  - 顶部按钮按身份切换：房主 `✕ 结束`；其他 `← 退出`
  - `handleLeave`：参赛者调 `seat leave`，观众直接跳 `/me`
  - 监听 WS `match_event type=end/force_end` 且非自己触发 → 弹 3s 倒计时 overlay → 跳 `/me`
  - `handleEnd` 后 `switchTab('/pages/me/index')`，不再停留
- `pages/join/index.tsx`：
  - 进页面未登录自动弹 `LoginSheet`
  - `matchApi.join` 传 `cloudUser.nickname` 覆盖号位原昵称
- `pages/join/index.scss`：`.join-btn` 非 primary 补齐 bg/color/border（白底白字修复）
- `pages/index/index.tsx`：点"🔗 加入联机房间"未登录弹登录；已登录才 navigate
- `pages/config/index.tsx`：
  - `online` 初始化 `!!cloudUser`（登录默认联机）
  - 未登录点切换 toggle → 弹 `LoginSheet`，登录成功后自动 `setOnline(true)`
- `pages/me/index.tsx`：
  - 历史记录加"云端 / 本地"Tab 切换
  - 云端条目点击跳 `/pages/match-detail?id=xxx`
  - `useDidShow` 刷新云端历史
- **新页** `pages/match-detail/`：比赛详情（摘要 + 玩家比分/stats + 完整事件日志，已撤销条目删除线）
- `app.config.ts`：注册 match-detail 路由
- `core/api/match.ts`：`seat` / `join` 加 `displayName` 参数；新增 `events` 拉全量事件
- `core/api/config.ts`：H5 下 API base 根据 `window.location.hostname` 推断（方便局域网同事测）
- `config/index.js`：Taro h5 devServer 绑 `0.0.0.0` + `allowedHosts: all`

### 服务端（NestJS）

- `src/match/match.service.ts`：
  - **新开房自动关旧房**：`create` 开头扫描该 user 所有未结束房 → `endByOwner('auto_closed_by_new_match')` → WS 广播 end 事件
  - `joinByCode` / `occupySlot` 加 `displayName?: string` 参数，非空则覆盖 `match_players.displayName`
  - `listMyMatches`：改为对每条返回完整 detail（含 `computed` / `timer`），前端列表能直接用
  - 新增 `listEvents(matchId)`
- `src/match/match.controller.ts`：`GET /v1/matches/:id/events`（参与者可查）；`/join` 和 `/seat` 接收 displayName
- `src/match/dto/match.dto.ts`：`JoinMatchDto` / `SeatActionDto` 加 `displayName`

### 管理后台

- `admin-web/src/pages/Dashboard.tsx`：加"进行中/等待中的房间"表格，5 秒自动刷新，房间码金色大字突出
