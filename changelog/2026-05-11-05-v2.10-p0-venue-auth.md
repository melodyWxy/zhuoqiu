# v2.10 P0：schema + 商家账号登录（双 client）

## 范围

P0：Prisma 建表 + 独立商家 JWT 登录，打下 v2.10 后续模块的地基。

## 服务端改动

### Prisma

- 新增 enum：`VenueAccountRole` / `VenueAccountStatus` / `VenueStatus` / `VenueApplicationStatus` / `VenueApplicationSource`
- 新增 model：
  - `VenueAccount`（商家账号，手机号唯一，role=owner/staff）
  - `Venue`（球房主体，owner 1:1 + staff 1:N）
  - `VenueApplication`（入驻申请，source=c_app|admin_web）
- `PhoneCodePurpose` 加 `venue_login`
- Migration: `20260511055835_add_venue_account_and_venue`

### 认证扩展

- `jwt-payload.ts` 加 `VenueAccountJwtPayload` / `VenueAccountRefreshPayload`，access payload 带 `type: venue_account` + `role` + `venueId` + **`client: 'admin_web' | 'c_app'`** claim
- `auth.service.ts` 加 `issueVenueAccountTokens` / `refreshVenueAccountAccessToken` / `verifyVenueAccountAccessToken`

### 新模块 `src/venue/`

- `venue.module.ts`：依赖 `AuthModule`
- `venue.service.ts`：`upsertByPhoneLogin`（首次登录自动建账号）
- `venue-auth.controller.ts`：
  - `POST /v1/venue-auth/sms/send`
  - `POST /v1/venue-auth/sms/verify`（入参 `client` 决定 token 的 client claim）
  - `POST /v1/venue-auth/refresh`
  - `POST /v1/venue-auth/logout`
  - `GET /v1/venue-auth/me`
- `venue-auth.guard.ts`：校验 venue_account JWT；读 `@RequireClient(...)` metadata 决定是否拦截
- `require-client.decorator.ts`：写入接口用 `@RequireClient('admin_web')` 限定，C 端 token 会被 `60010 VENUE_ONLY_ADMIN_WEB` 拒绝（给前端引导文案用）
- `current-venue-account.decorator.ts`：controller 里直接拿 payload

### 错误码

`60001 VENUE_NOT_FOUND` / `60002 VENUE_APPLICATION_NOT_FOUND` / `60003 VENUE_APPLICATION_STATE_INVALID` / `60010 VENUE_ONLY_ADMIN_WEB`

### Seed

加开发用商家账号：`phone=13900000001`，默认昵称"开发用商家"，未绑定 venue。

## 验证（本地 curl）

```text
✓ POST /v1/venue-auth/sms/send                   → 200 + devHint
✓ POST /v1/venue-auth/sms/verify (admin_web)     → accessToken(client=admin_web) + refreshToken
✓ POST /v1/venue-auth/sms/verify (c_app)         → accessToken(client=c_app)
✓ GET  /v1/venue-auth/me (admin_web token)       → account + client:admin_web + venue:null
✓ GET  /v1/venue-auth/me (c_app token)           → account + client:c_app
✓ GET  /v1/venue-auth/me (无 token)              → 10002
```

JWT payload 解码确认：
- `type=venue_account` ✓
- `sub=va_xxxxxxxx` ✓
- `role=owner` ✓
- `venueId=null`（未绑定） ✓
- `client=admin_web` 或 `c_app` ✓
