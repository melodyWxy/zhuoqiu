# P1 补丁 2 球房编辑/审核/赛事详情侧的省市区显示与录入补齐

## 动机
真人测试时用户发现「编辑店铺资料」页没有省市区 picker —— 只加了入驻表单，
没补编辑页。盘点一遍发现还有 6 处其他链路的展示和类型遗漏：

| 文件 | 问题 |
|---|---|
| `admin-web/Profile.tsx` | 编辑店铺没省市区 picker（**用户发现**） |
| `admin-web/Overview.tsx` | 商家概览卡只显示详细地址，不带省市区 |
| `admin-web/Venues/ApplicationDetail.tsx` | 平台 admin 审核页看不到商家选的省市区，盲审风险 |
| `admin-web/VenueMerchant/ApplyStatus.tsx` | 商家自己看「我的申请状态」也只显示详细地址 |
| `billiards-score/tournament-detail/index.tsx` | C 端赛事详情主办球房只显示详细地址 |
| `billiards-score/core/api/venue.ts` `TournamentDetailPublic` | 类型缺 city/province/district，P3 跨城判断会卡 TS |
| `admin-web/api/venue.ts` `me.venue` 类型 | 同样缺三字段 |

参考 PRD：[`prd/v2.21-nearby-by-city.md`](../prd/v2.21-nearby-by-city.md) §3.3。
对话决策：审核页用「单独一行省/市/区 + 单独一行详细地址」，让人工审核员一眼判断
两者是否自相矛盾。

## PRD / 设计变化
无 PRD 改动，纯实现补齐。

## 代码变化

### Admin-web
- `pages/VenueMerchant/Profile.tsx`：
  - 加 antd `<Cascader>` 字段「所在地区」+ 必填校验（length=3）
  - 拉 `/v1/regions` 进 state；初始化时把 `full.province / city / district` 三个一起塞回 form
  - 历史 venue 没全的话 region 给空数组让 placeholder 强制商家选一次
  - 提交时拆数组成三字段，跟着 patch 一起 PATCH
  - 详细地址 tooltip「不用重复省市区，只填街道及门牌号」
- `pages/VenueMerchant/Overview.tsx`：
  - `VenueMeState.venue` 加 province/city/district
  - 「地址」描述项改为 4 段拼接
- `pages/VenueMerchant/ApplyStatus.tsx`：
  - 「地址」描述项改 4 段拼接（`payloadJson` 已带新字段）
- `pages/Venues/ApplicationDetail.tsx`：
  - 「地址」一项拆成「所在地区」+「详细地址」两个独立 row；历史无三字段时显示
    「— （历史数据，未选）」提示
- `api/venue.ts`：
  - `UpdateVenuePayload` 加 province/city/district
  - `VenuePublic` 加 province/city/district + lat/lng
  - `venueAuthApi.me` 返回类型 `venue` 加三字段

### 服务端
- `venue/tournament.service.ts` `detailPublicById`：venue select 从 `id/name/address/city/coverImage`
  扩到含 province/district，给 c 端赛事详情完整 4 段拼接

### C 端 (billiards-score)
- `core/api/venue.ts`：`TournamentDetailPublic.venue` 类型加 province/city/district
- `pages/tournament-detail/index.tsx`：主办球房地址改 4 段拼接

## 遗留问题 / 已知限制
- 历史 venue district = null 的情况下，商家进编辑页会被强制选一次省/市/区才能保存。
  这是预期行为（让数据更规范），但 UX 上等于强迫商家更新一次资料 —— 可以等真上量后
  看用户反馈再决定要不要降级为非必填。
- Admin-web 没做平台 super_admin 直接编辑 venue 的入口，所以平台运营手填修复 city
  这条路目前还是没接通。等真有这种修复需求再加 admin endpoint。
- ApplyStatus 顶部 `setVenue` 状态有个内联类型已经过期（缺三字段），不影响功能
  （TS 接受 excess properties），后续重构再清理。
