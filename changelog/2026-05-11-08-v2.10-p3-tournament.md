# v2.10 P3：赛事 CRUD + 报名全链路

承接 P0-P2 的球房能力，P3 把赛事从 0 到"报名满员可关闭报名"的完整闭环跑通。

```
商家（admin_web）→ 新建赛事（draft）→ 发布（registering）
                                          │
                                          ▼
C 端用户 → 赛事列表 → 详情 → 报名确认 → confirmed
                                          │
                                          ▼
商家 → 查报名名单（含手机号）→ 满员或到点 → 关闭报名（registration_closed）
```

bracket 生成 + 开赛在 **P4** 落地。

## 服务端

### Prisma

新表 + 4 enum（migration `20260511071404_add_tournament`）：
- `Tournament`：标题 / gameType / format / rulesJson / 人数上下限 / 报名期 / 开赛时间 / 奖励描述
- `TournamentRegistration`：userId + displayName + phone 快照，`@@unique([tournamentId, userId])`
- `TournamentBracketMatch`：round + slotInRound + playerA/B + winner + matchId（关联 Match）
- enums：`TournamentFormat` / `TournamentStatus` / `TournamentRegistrationStatus` / `BracketMatchStatus`

### TournamentService

- 商家：`createDraft` / `update` / `publish` / `cancel` / `closeRegistration`
- 验证：minPlayers ≤ maxPlayers，regEnd > regStart，matchStart ≥ regEnd
- 用户：`register`（事务内校验状态+名额，支持 withdrawn 后重报）/ `withdraw` / `myRegistration` / `myTournaments`
- 商家踢人：`kick` → status=disqualified
- `findOwn(tournamentId, accountId)`：确保只有绑定同一 venue 的商家账号能操作自家赛事（61003 隔离）
- 读：`listPublic`（匿名；默认过滤 draft）/ `detailPublic` / `registrationsPublic`（只返回昵称/序号）

### Controllers

- `TournamentMerchantController` `/v1/venue/tournaments/*`：VenueAuthGuard + `@RequireClient('admin_web')`（写入接口）
- `TournamentPublicController` `/v1/tournaments/*`：list/detail/registrations 匿名；register/withdraw/my-registration 需 UserAuthGuard
- `MyTournamentsController` `/v1/me/tournaments`

### 错误码

`61001 TOURNAMENT_NOT_FOUND` / `61002 STATE_INVALID` / `61003 NOT_OWNER` / `61004 REGISTRATION_CLOSED` / `61005 REGISTRATION_FULL` / `61006 ALREADY_REGISTERED` / `61007 REGISTRATION_NOT_FOUND`
