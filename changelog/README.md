# 迭代日志

每次需求变更按日期 + 序号新建一个 markdown：`YYYY-MM-DD-NN-短标题.md`。

**工作流**（见记忆中的 `prd-first-workflow`）：
1. 用户提需求 → 先改 PRD（`prd/billiards-match-app-prd.md`）和设计稿（`ux/billiards-app-design-v1.md`, `ux/index.html`）
2. 给用户展示变更点 → 等确认
3. 确认后再写代码
4. 落地后回到本目录追加本次日志

**日志模板**（见 `_template.md`）：每条包含 动机 / PRD & 设计变化 / 代码变化 / 验证步骤。
