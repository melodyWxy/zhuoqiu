# 小程序审核合规 - 登录/绑定文案去「微信」字样

## 动机

微信小程序提交审核被驳回：登录页/弹窗（调用手机号快速验证组件的前置页面）里有
「微信授权手机号」等含「微信」字样的用户可见文案，被判定为「混淆官方元素」。
需去掉这些字样，改成中性表述（如「手机号快捷登录」）。

## 代码变化（只改用户可见文案，代码注释 / 哨兵常量 / 隐私政策不动）

### `billiards-score/src/components/LoginSheet/index.tsx`
- 手机号授权按钮：`📱 微信授权手机号` → `📱 手机号快捷登录`
- 加载标题：`正在通过微信登录…` → `正在登录…`
- 手机号授权说明：`获取已绑定微信的手机号` → `获取你的手机号`
- 隐私同意文案：`收集手机号或微信账号信息` → `收集手机号或账号信息`
- 资料补全说明：去掉「点击下方按钮可使用你的微信资料」
- 头像提示：`点击选择微信头像` → `点击选择头像`
- 昵称 placeholder：`点击使用微信昵称` → `点击输入昵称`
- 登录失败 toast：`微信登录失败，请重试` → `登录失败，请重试`（×2）

### `billiards-score/src/components/BindPhoneSheet/index.tsx`
- 手机号授权按钮：`📱 微信授权手机号` → `📱 获取手机号`
- 绑定说明：`授权使用与微信绑定的手机号` → `使用本机号码快捷绑定`
- 合并冲突说明：`战绩和微信/抖音绑定` → `战绩和账号绑定`

## 不动的部分（说明）

- 代码注释、`DEFAULT_WECHAT_NICKNAME = '微信用户'`（detection 哨兵，不展示给用户）。
- 隐私政策 `pages/legal/content.ts` 中的「微信」：依法须披露第三方数据来源，保留。
- `getPhoneNumber` / `open-type` 等原生能力逻辑不变。

## 待确认（非本次代码范围）

- 默认昵称「微信用户」：`server/src` 未搜到该字面量，应不会显示成玩家名；若线上确有
  用户名显示为「微信用户」，需在**服务端/数据库默认值**处改（小程序代码改不到）。

## 部署

- weapp 重打 `build:weapp:prod` dist（已完成），用微信开发者工具上传后重新提交审核。
- 纯前端文案改动，server 无需变更。
