# v2.20：Dockerfile 切 npm 源 + Prisma 引擎源到国内镜像

## 背景

国内服务器 `npm ci` 走默认源 `registry.npmjs.org` 慢；server 还要从
`binaries.prisma.sh` 下 4 个 Prisma 引擎共 200MB+，是主要瓶颈。

## 改动

### 三个 Dockerfile 的 `npm ci` 前都加 `npm config set registry https://registry.npmmirror.com`

- [server/Dockerfile](server/Dockerfile)
- [billiards-score/Dockerfile](billiards-score/Dockerfile)
- [admin-web/Dockerfile](admin-web/Dockerfile)

### server 额外配 Prisma 引擎镜像

```dockerfile
ENV PRISMA_ENGINES_MIRROR=https://registry.npmmirror.com/-/binary/prisma
ENV PRISMA_BINARIES_MIRROR=https://registry.npmmirror.com/-/binary/prisma
```

放在 `npm ci` 之前。Prisma 的 `@prisma/client` postinstall 会读这俩
环境变量决定从哪下引擎。

## 效果

- h5 / admin `npm ci`：5-10 分钟 → 1-2 分钟
- server `npm ci`：10+ 分钟 → 2-3 分钟（Prisma 引擎从国内 CDN 拉）

## 注意

- 海外服务器 build 会变慢，必要时把 `npm config set` 注释掉
- npmmirror 的 prisma binary 可能比官方延迟几小时同步，极少数情况会
  缺新版本——遇到再 fallback 到官方源
