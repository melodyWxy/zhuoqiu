# 二期批次 2 · 管理后台 UI 联调成功

## 动机

服务端已实装 30 个端点，现在做 admin-web 联调：从登录走到房间详情、用户封禁、审计查询、系统设置，把"后端 + 管理端"这对闭环跑通。

## PRD / 设计变化

完全按 `ux/v2/admin-shared-match.md` 设计实现，无新增章节。

## 代码变化

### 新增 `admin-web/`

- **脚手架**：Vite 5 + React 18 + TypeScript + AntD 5 + `@ant-design/pro-components`
- **Vite 配置**：`/v1/*` 代理到 `http://localhost:3001`，免跨域配置
- **API 客户端**（`src/api/`）：
  - `client.ts`：axios 实例，请求拦截器注入 `Authorization`；响应拦截器解包 `{code,data,traceId}`、业务错误 message.error、401 自动 refresh + 重试、过期跳回登录
  - `auth.ts` / `matches.ts` / `users.ts` / `misc.ts`（analytics + audit + settings）
- **状态**（`src/stores/auth.ts`）：Zustand + localStorage 持久化 access/refresh/account
- **页面**（`src/pages/`）：
  - `Login.tsx`：账号密码登录，登录后跳首页
  - `Dashboard.tsx`：4 个核心指标（在线房间 / 今日创建 / 今日结束 / 今日新注册）+ 同比指示 + 占位图表
  - `Matches/List.tsx`：筛选（房间码/昵称/手机、状态多选、类型）+ 分页表格
  - `Matches/Detail.tsx`：基本信息 + 玩家比分（九球显示 💎🏅👑✅ 统计；中八显示胜局）+ 完整事件日志（含已撤销条目，删除线）+ 强制暂停/结束按钮（弹确认 + 填原因）
  - `Users/List.tsx`：头像 + 昵称 + 脱敏手机号 + 状态 tag
  - `Users/Detail.tsx`：profile + 封禁/解封 + 微信/抖音绑定 tabs
  - `Audit.tsx`：按操作类型/目标 ID 筛选的审计日志
  - `Settings.tsx`：8 个系统设置字段（super_admin 可改，其他只读）
- **布局**（`src/components/AppLayout.tsx`）：`ProLayout` 带侧栏 + 顶栏 + 用户菜单 + 角色 Tag + 登录守卫
- **类型**（`src/types/index.ts`）：与后端响应对齐的 15 个 TS 接口

### 后端修复

- `server/src/admin/matches-admin.controller.ts`：`GET /admin/matches/:id` 原来返回 Prisma 原始字段（timerStartedAt 扁平），改为复用 `MatchService.detail`（含聚合 `timer` / `computed` 字段），保持与 C 端 detail 结构一致。前端崩溃 `Cannot read properties of undefined (reading 'accumulatedMs')` 修复。
