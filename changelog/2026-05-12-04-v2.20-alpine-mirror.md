# v2.20：Dockerfile 用阿里云 Alpine 镜像源

## 背景

国内服务器 `apk add python3 make g++ ...` 走 Alpine 默认源 `dl-cdn.alpinelinux.org` 极慢（g++ 几十 MB，单步可能 10+ 分钟）。

## 改动

在所有 `apk add` 之前加一行 `sed` 把 repo 切到 `mirrors.aliyun.com`：

- [server/Dockerfile](server/Dockerfile) builder + runner stage 各一处
- [billiards-score/Dockerfile](billiards-score/Dockerfile) builder stage 一处
- `admin-web/Dockerfile` 不装系统包，无需改

```dockerfile
RUN sed -i 's|dl-cdn.alpinelinux.org|mirrors.aliyun.com|g' /etc/apk/repositories \
    && apk add --no-cache <packages>
```

## 效果

`apk add` 步骤通常从 10+ 分钟降到 1-2 分钟。

## 注意

- 仅影响构建期，不影响运行时
- 海外服务器 build 反而会变慢，必要时把 sed 行注释掉
