# v2.10 P5：bracket ↔ match 联动 + 现场控台

赛事终于跑完了完整闭环：**报名 → 开赛 → 对阵 → 双方记分 → 自动推进 → 冠军**。

## 服务端

### Schema

- `Match.venueId` + `TournamentBracketMatch.matchId` 双向关联（matchId 侧 unique + FK，Match 侧声明反向 relation `tournamentBracketMatch`）
- `Venue.matches[]` 反向关系
- migration: `20260511092047_add_match_venue_bracket_link`

### BracketService（新）

- `openBracketMatch(tournamentId, bmId, accountId)`：
  - 校验 bracket.status=ready、双方就位、未开赛、商家属于同 venue
  - 生成 6 位房间码 + 创建 `Match`（owner=playerA.userId, type=赛事项目, venueId, rules 快照）
  - 两个 slot 自动占位 playerA/B（displayName + userId）
  - 绑 `bracket.matchId`，状态 → `in_progress`
- `manualWalkover(tournamentId, bmId, accountId, winnerSide)`：判一方胜，bracket 立即推进
- `advanceWinnerToNextRound` / `maybeCompleteTournament`：公用推进逻辑

### Match.doEnd hook（`venue/bracket-resolve.ts`）

- 找该 match 的 bracket；复用 `applyNineBallEvent` / `applyEightBallEvent` state-machine 算 slot 1 / slot 2 分数
- 分高者的 registration → `bracket.winnerRegistrationId`，status → `completed`
- 调用 `advanceWinnerToNextRound` 填下一轮；若最后一轮全部完成 → 赛事 `completed`
- 作为纯函数导出，通过接口回调 Prisma 操作；避免 venue ↔ match 模块循环依赖

### Controller

- `POST /v1/venue/tournaments/:id/bracket/:bmId/start`（@RequireClient admin_web）
- `POST /v1/venue/tournaments/:id/bracket/:bmId/walkover` body `{winnerSide: 'A'|'B'}`

## Admin-web

### TournamentDetail.tsx

- 对阵卡片可点击（ready / in_progress / 一方就位的 pending）
- 弹 `BracketActionModal`：
  - 标题：`Round N slot M`
  - VS 面板：双方昵称 + seed
  - 操作按钮：
    - `ready` → 🚀 开始比赛（建 Match，自动新窗口打开 C 端联机页 `pages/nine-ball|eight-ball`）
    - `in_progress` → "打开记分页（新窗口）"
    - `ready/pending 带人` → 判 A/B 胜（walkover）
- 正在进行中的对阵卡蓝色边框 + 蓝色状态
- api/venue.ts 新增 `startBracketMatch` / `walkover`

## C 端

### tournament-detail

- "我的对阵"卡点击行为按状态区分：
  - `in_progress` + `matchId` → 直接 navigateTo 对应 nine-ball / eight-ball 联机页（带 `matchId` 和 `role=player`）
  - `ready` → toast "等商家在控台开赛"
  - `completed` → toast "该场已结束"
- 卡上"进行中"提示改成"🔵 进行中 · 点击进入记分 →"

## E2E 验证（curl）

```text
✓ 商家开赛 ready 的 R1 bracket → 返回 matchId + code
✓ bracket.status = in_progress，matchId 已绑定
✓ admin force_end match（跳过真实记分）
✓ bracket 自动 completed，winner=playerA（平局兜底）
✓ R2 对应 slot 自动填入 winner，双方齐备 → ready
✓ 下一个 R1 ready 可继续开，走到决赛 → 赛事 completed
```

真实流程（demo）：
1. 商家 bracket Tab 点对阵卡 → 弹 Modal → 🚀 开始比赛
2. 新窗口打开联机大屏（`http://host:3000/#/pages/nine-ball/index?matchId=...`）
3. 双方用各自手机登录 C 端（用户 = 报名的手机号）→ 看到自己"我的对阵"进入
4. 任意一方手机记分（已确保 matchPlayer.userId 绑定）
5. 比赛结束（某人到目标分或 admin force_end）→ bracket 自动推进
