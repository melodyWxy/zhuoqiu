# P1 补丁 球房入驻明确选省市区，腾讯地图退化为只算 lat/lng

## 动机
P1 上线方案是「商家只填自由文本地址，后端用腾讯地图正向解析反猜城市」。
用户挑出问题：**地址写得不规范就解析失败，城市归属丢失，店在「附近=同城」搜索里就找不到了**。

修正方案：
1. 商家入驻时必须**明确**通过省/市/区三级 picker 选所在地区，作为权威值落库
2. 腾讯地图退化为辅助工具，只用来填同城距离排序需要的 lat/lng
3. 数据源用腾讯地图的「行政区划查询 WebService」实时拉，服务端缓存 24h，前端各自缓存 7 天，
   背靠民政部数据，唯一权威源、零静态维护

参考 PRD：[`prd/v2.21-nearby-by-city.md`](../prd/v2.21-nearby-by-city.md) §3.6 / §5。

## PRD / 设计变化
PRD §3.6「城市选择器」原文是「MVP 阶段硬编码省 → 市两级菜单」。本期升级为：

- 三级（省 / 市 / 区，全部必填）
- 数据走 `/v1/regions` 实时拉，不硬编码
- 直辖市补层「市级镜像」让前端 cascader 看到统一三级结构

后续 PR 里 PRD 文档会同步更新。

## 代码变化

### 服务端：新模块 `server/src/regions/`
- `regions.service.ts`：调腾讯地图 `/ws/district/v1/list`，把「分层 + cidx 索引」结构转嵌套
  树。直辖市 layer-1 直接是区，普通省 layer-1 是市；判定方法是「行政码后 4 位是否
  全 0」，对直辖市补层「市级 = 省自身」。缓存 24h；过期 1 周内拉新失败仍用旧缓存
  兜底，避免单点抖动让入驻流程整个挂掉。`inflight` Promise 合并并发请求。
- `regions.controller.ts`：`GET /v1/regions`，公开。
- `regions.module.ts` + 接进 `app.module.ts`。

### 服务端：DB schema
- `prisma/schema.prisma`：`Venue` 加 `province String? @db.VarChar(32)` + `district String? @db.VarChar(32)`
- `prisma/migrations/20260529071132_add_venue_province_district/`

### 服务端：DTO

### 服务端：service 逻辑改写
- `venue-application.service.ts`：`approve` 流程
  - 省/市/区直接用 payload 的值落库（权威）
  - 拼成 fullAddress 给腾讯地图 `resolveAddress`，**只取 lat/lng**，city/province 不再依赖它
- `venue.service.ts`：`updateOwnVenue`

- `venue.service.ts` `VENUE_PUBLIC_SELECT`：加 province/district
- `venue-auth.controller.ts` `me` 响应里 `venue` 加 province/city/district

## 遗留问题 / 已知限制
- **历史 venue 的 district = null**：自动回填脚本不动 district（腾讯反向解析对区的命中率不稳定），
  老店要么商家自己进编辑页改一下，要么平台运营手动后台补。新入驻的全部强制三级。
- **腾讯地图 /ws/district/v1/list 配额**：免费每日 1 万次，缓存 24h 后我们一天 1 次。
  极端情况：服务实例发生重启风暴 → 还是有兜底（陈旧缓存可用 1 周）+ 单飞合并。
- **直辖市的「市级」是 picker 里点一下就过的占位项**：北京 → 选「北京市」（同名）→ 选区。
  UX 上可以未来优化为「直辖市自动跳到选区」，本期先按统一三级走，避免特判。
- **行政区划变更**：撤县设市/合并/改名等动作发生后，新 venue 选到的 name 字符串可能跟
  旧 venue 落库的 name 不一致。同城搜索是 string 严格相等比对 → 旧店暂时搜不到。
  P3/P4 之后如真碰上这类问题，再切到「按 6 位行政码比对」的口径。
- **admin-web 没有「查看历史申请」时把 region 三字段铺开展示**：申请详情页 / 申请列表页
  的 payload 显示需要 review，本期没动 admin 审核侧的 UI（数据已经传过去了）。
