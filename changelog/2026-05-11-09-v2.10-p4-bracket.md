# v2.10 P4：bracket 生成 + 赛程可视化

承接 P3（赛事 CRUD + 报名），P4 把"报名截止 → 生成赛程 → 开赛"这一步跑通。此时只画树、自动处理 BYE；match 与赛程的联动（开局 / winner 回填）留到 P5。

## 服务端

### 新增 `src/venue/bracket-utils.ts`

- `makeSeedOrder(k)`：递归生成经典 seed 分布（保证头部种子晚相遇）
  - k=3 → `[1, 8, 4, 5, 2, 7, 3, 6]`
- `bracketPower(n)` / `planBracket(n)`：补 BYE 到 2^⌈log2 n⌉；返回 `firstRoundSeeds`（seed 号 > n 则为 null = BYE）

### `TournamentService.startTournament(id, accountId)`

事务内：
1. 校验状态 `registering | registration_closed`、赛制 `single_elim`、人数 ≥ `minPlayers`
2. 按 `registeredAt` 给报名者分配 seed `1..n`
3. 按 `planBracket(n)` 生成首轮 `matchesPerRound[0]` 对阵：
   - 双方都有人 → `status=ready`
   - 一方 null → `status=walkover`，winner 填非 null 方
4. 其余轮次全部 `pending`（空 player）
5. 首轮 walkover 自动把 winner 填入下一轮对应位置（偶 slot→A，奇 slot→B）；若下一轮双方就位即 `ready`
6. Tournament 状态置 `in_progress`

每次重新调用会清空旧 bracket（支持重开赛；生产可加锁）。

### Controller

- 商家：`POST /v1/venue/tournaments/:id/start`（`@RequireClient('admin_web')`）+ `GET /v1/venue/tournaments/:id/bracket`
- 公共：`GET /v1/tournaments/:id/bracket`

### `getBracket`

返回 `rounds: [{round, matches[]}]`，matches include `playerA/B/winner`（seed + displayName + userId）。
