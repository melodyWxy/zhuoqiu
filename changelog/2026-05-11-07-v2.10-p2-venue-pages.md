# v2.10 P2：店铺资料编辑 + C 端球房发现

P1 完成商家入驻闭环后，P2 把"已入驻商家"和"C 端用户"两侧的球房视图接上。

## 服务端

- `VenueService` 加 `listPublic` / `getPublic`（只返回 status=active 的公开字段）
- `VenueService.updateOwnVenue(accountId, patch)`：校验 role=owner + venueId 非空；支持部分字段更新
- 新 controller `venue.controller.ts`：
  - `GET /v1/venues`（公开，支持 keyword 模糊 + 分页）
  - `GET /v1/venues/:id`（公开）
  - `PATCH /v1/venue/me/venue`（VenueAuthGuard + `@RequireClient('admin_web')`；c_app token 调会返回 60010）
- DTO `UpdateVenueDto`：全部字段 optional，复用 OpenHoursItemDto 校验

## Admin-web 商家侧

- `pages/VenueMerchant/Overview.tsx` `/venue/overview`：店铺概览（ID / 地址 / 状态 / 台桌数）+ 快捷操作（赛事管理/控台占位 P3-P5）
- `pages/VenueMerchant/Profile.tsx` `/venue/profile`：店铺资料编辑表单（名称 / 地址 / 电话 / 台桌数 / 营业时间 / 简介 / 封面图），封面上传走 `/v1/uploads?category=venue-cover`
- `VenueLogin` / `Apply` / `ApplyStatus` 的已绑定 venue 跳转统一指向 `/venue/overview`（替换旧的 `/apply/status` 占位）
- `api/venue.ts` 加 `venueMyApi.update` + `VenuePublic` 类型

## C 端 Taro

- `core/api/venue.ts` 加 `venuesPublicApi.list/detail`（匿名可访问）
- 新页 `/pages/venues/`：列表，搜索 + 下拉刷新；卡片展示封面 / 店名 / 地址 / 台桌数 / 营业时间
- 新页 `/pages/venue-detail/`：详情，封面 + 店名 + 地址（点击复制）+ 电话（点击拨号 / 复制）+ 营业时间（相同 / 按天）+ 简介
  - 尾部预留"赛事 P3"和"本月排行 v2.11"卡片占位
- 首页 `/pages/index/` 加 `🏢 发现球房` 卡片 → 跳列表
- `app.config.ts` 注册两个新路由

## 验证（curl）

```text
✓ GET /v1/venues              → 返回 active 球房列表
✓ GET /v1/venues?keyword=开发 → 按名称/地址 contains 模糊
✓ GET /v1/venues/:id          → 公开字段
✓ PATCH /venue/me/venue (admin_web) → 改掉 description 成功
✓ PATCH /venue/me/venue (c_app)     → 60010 "此操作仅可在管理后台进行"
```
