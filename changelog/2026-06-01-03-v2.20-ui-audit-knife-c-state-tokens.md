# UI 审计·刀 C - 状态 / 层级 token 落地

## 动机
`ux/v2.20-ui-audit.md` 立的第 3 把刀。审计发现：

- disabled opacity 三档混用：0.4（八球 / 九球 op-btn）/ 0.5（join /
  match-detail）/ 0.55（venue-login send-btn）
- 圆角 5 档：8 / 10 / 12 / 14 / 16px 散落
- 字号无系统：11/12/13/14/15/16/17/18/20/22/24px 全员上岗
- 5 个弹窗组件的遮罩 BG 4 个用 `rgba(0,0,0,0.55)`，1 个 mhs-mask 用 0.6（差
  0.05 透明度，看不出来但破坏了「同一组件类目」的一致性）

刀 C 不是一次性把所有散点都收齐，而是**先布点 token + 渐进替换**，在弹窗 / 我页 /
球房 / 八球 / 九球 / 球房登录这几个高频文件里做样本替换，让后续写新 UI 时
有现成 token 抓手。

## PRD / 设计变化
- `ux/v2.20-ui-audit.md` 刀 C 一节
- 新增 token（写进 `styles/global.scss` `@mixin color-tokens` 里，与
  `--accent` 等同档管理）：
  - `--opacity-disabled: 0.4`
  - `--opacity-active: 0.85`
  - `--radius-sm: 8px` / `--radius-md: 12px` / `--radius-lg: 16px`
  - `--font-size-caption: 12px` / `--font-size-body: 14px` /
    `--font-size-title: 16px` / `--font-size-heading: 20px`

字号 token 本轮**只定义不替换**——字号语义需要逐文件 case-by-case 判断，留
给后续刀 D 之后做。

## 工程注记：mixin 写法

PRD 原计划在 `global.scss` 加 `@mixin text-truncate-1 / text-truncate-2 /
modal-mask`。落地时发现：本项目所有页面 / 组件 scss 都没有 `@import` /
`@use` 任何文件，sass 也没配 `additionalData` 注入，**跨文件 `@include` 不
通**。临时补 `@use` 链路会改动 30+ 文件的 scss 头部，与本刀「渐进、低风险」
的目标冲突。

折中方案：在 `global.scss` 把这些片段写成**复用注释片段**（不是 mixin），
注明改动这些片段时要同步 5 个弹窗组件。token (CSS 变量) 走 `page` / `body`
继承，不受这个限制，正常用 `var(--radius-sm)`。

## 代码变化

### Token 定义入口
- `styles/global.scss`：`@mixin color-tokens` 内追加 9 个新 token；末尾加
  「复用片段」注释（mask / truncate 模板）

### Disabled opacity 统一 → `var(--opacity-disabled)`
| 文件 | 原值 |
|------|------|
| `pages/eight-ball/index.scss` `.op-btn.is-disabled` | `0.4` |
| `pages/nine-ball/index.scss` `.op-btn.is-disabled` | `0.4` |
| `pages/join/index.scss` `[disabled]` | `0.5` → 0.4 |
| `pages/venue-login/index.scss` `.vl-send-btn[disabled]` | `0.55` → 0.4 |

### 圆角样本替换 → `var(--radius-sm)` / `var(--radius-md)`
- 5 个弹窗组件的 `border-radius: 8px;` 全部 → `var(--radius-sm)`
  - `InputModal` `LoginSheet` `AvatarPickerModal` `FeedbackModal`
    `MatchHistorySheet`（命中 ≈ 8 处，靠脚本批量替换 + grep 复核）
- `pages/me/index.scss` `.section`、`.venue-mode-card` 12px → `var(--radius-md)`
- `pages/venues/index.scss` `.vp-card` 12px → `var(--radius-md)`

### 弹窗遮罩透明度统一
- `MatchHistorySheet/index.scss` `.mhs-mask` BG 0.6 → 0.55，与其他 4 个弹窗
  齐平

## 故意不替换
- **`opacity: 0.5/0.55` 标记「撤销过的事件」/「划线状态」**
  （`match-detail` `.undone`、`MatchHistorySheet` `.is-undone`）：语义不是
  disabled 而是 deemphasized，独立设计语言，保留字面量。
- **`opacity: 0.85/0.7` 的 `:active` 反馈**：是交互态，不属于 disabled token。
- **8/12/16 以外的 radius**（10 / 14 / 20 / 50% / 999px）：没有对应 token，
  本轮不引入，保留字面量。
- **字号**：定义了 token 但没替换；字号有强语义（heading vs title vs body），
  替换前需要 case-by-case 评估，留给后续。
- **`PageHeader` `GameToolbar` 等小组件**：圆角 / opacity 都已经合理，不为
  token 化而 token 化。

## 遗留问题 / 已知限制
- **跨文件 mixin 没接通**：未来若想用 `@include text-truncate-1`，需要把
  各 scss 文件统一 `@use '../../styles/global' as *;`，工作量大，留给
  专项重构
- **字号 token 定义了未替换**：当心散落字号 11/13/15/17/18 等没对应 token
  的值；本轮不强行 token 化
- **ConnectionBanner 状态色**：与刀 B 同一原因（contrast-tuned variants），
  未 token 化
- **`opacity: 0.6` 在 me 页 `.placeholder-text`**：不是 disabled 也不是
  undone，是「占位文字」，保留
