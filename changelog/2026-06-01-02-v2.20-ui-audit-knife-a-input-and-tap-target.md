# UI 审计·刀 A - 输入框高度 + 触达统一

## 动机
- `InputModal` `LoginSheet` 输入框 `height: 48px`，介于「能点」与「舒适」之间
- 球房列表 `vp-region-cell` 省市区筛选格 `padding: 8px 10px; font-size: 13px;`
  实测高度仅 ≈ 32px，明显偏矮，用户原话「省市区的选择框拉长一点」
- `vp-search-input` 没有显式 height，靠 padding 撑高，placeholder 偏移
- `vp-clear-btn` `font-size: 12px; padding: 6px 8px` 触达偏小
- 几个主操作按钮（弹窗 OK/取消、avatar cancel、feedback confirm）padding 12 +
  font 15 ≈ 42，差 2px 触不到「移动端 ≥44px」的标准
- `vl-field` `va-field` `va-region-cell` 都靠 padding 撑高度，没用 `height`

遵循 [[feedback-weapp-input-height]]：weapp 输入框调高度用 `height`，不堆
竖向 padding。

## PRD / 设计变化
- `ux/v2.20-ui-audit.md` 刀 A 一节
- 高度档定下来：
  - **弹窗输入** 56px（首选触达）
  - **页面内输入** 48px（信息密度高，节省屏幕空间）
  - **搜索 / 选择 cell** 44px（与搜索按钮齐高）
  - **主操作按钮** padding 14（≈ 47px）
- 不动字号系统（留给刀 C）；不动 `:active` 反馈

## 代码变化

### 弹窗输入 → 56px
- `components/InputModal/index.scss`：`.input-modal-field` height 48 → 56；
  `.input-modal-btn` padding 12 → 14
- `components/LoginSheet/index.scss`：`.login-sheet-field` height 48 → 56

### 弹窗主操作按钮 → padding 14
- `components/FeedbackModal/index.scss`：`.feedback-modal-btn` padding 12 → 14
- `components/AvatarPickerModal/index.scss`：`.avatar-picker-cancel` padding 12 → 14

### 球房列表 → 44px 一档
- `pages/venues/index.scss`：
  - `.vp-search-input`：加 `height: 44px; box-sizing: border-box; padding: 0 14px`，
    去掉之前靠 `padding: 10px 14px` 撑高度
  - `.vp-search-btn`：加 `height: 44px`，padding 0 16 → 0 18，与 input 齐高
  - `.vp-region-cell`：加 `height: 44px; display: flex` 居中；padding 8/10 → 0/12；
    font 13 → 14（用户「省市区拉长一点」直接落地点）
  - `.vp-clear-btn`：font 12 → 13；padding 6/8 → 10/12

### 球房入驻表单 → 48px 一档
- `pages/venue-apply/index.scss`：
  - `.va-field`：加 `height: 48px; box-sizing`；padding 10/12 → 0/12
  - `.va-region-cell`：加 `height: 48px; flex 居中`；padding 10/12 → 0/12
  - `.va-textarea`：min-height 60 → 88，并明确 `padding: 10px 12px; height: auto`，
    避免被 `.va-field` 的 height: 48 覆盖压扁

### 球房登录 → 48px + 验证码按钮齐高
- `pages/venue-login/index.scss`：
  - `.vl-field`：加 `height: 48px`；padding 10/12 → 0/12
  - `.vl-send-btn`：加 `height: 48px; line-height: 46px`，原来 `line-height: 36px`
    与 vl-field 横排错位，改后齐高
  - `.vl-login-btn`：padding 10 → 14，触达 ≥ 44px

## 故意不改
- **`config/index.scss`**：`.input-item` 是行容器（label + input 合在一起），
  整行 `padding: 12px` ≈ 48px 已 OK；裸 `.input-field` 不需要单独 height。
- **`PageHeader` `GameToolbar` 的小图标按钮**（32×32）：现有 padding 后点击区
  ≥ 32×32，业务上是次要操作，本轮不强制提到 44。
- **slot 渐变次色 / ConnectionBanner 状态色**：与刀 A 无关。

## 验证步骤
- [x] grep 复核：所有 input / cell / 主按钮都有 height 或 padding ≥ 14
- [ ] H5 真机 + 微信开发者工具：
  - 输入弹窗 / 登录 sheet：输入区舒适、不挤
  - 球房列表 vp-search 与 vp-region-cell 横向齐高
  - 省市区 picker 三格在 iPhone SE (375pt) 不溢出
  - 球房入驻表单逐项无塌陷
  - 球房登录验证码按钮与输入框横向齐高
- [ ] weapp：输入聚焦键盘弹起后 placeholder 不偏移（用 height 不用 padding 的关键检验点）

## 遗留问题 / 已知限制
- 球房入驻 textarea min-height 提到 88，多行体验更好；但 padding 与单行 field
  的 0 不一致（textarea 内容需要内边距），是有意的差异
- 主页热门列表 `.hot-item` 高度未拉齐到 44，因为信息密度需求不同（一屏要塞
  多张），保留现状
- `config` 的「自定义分数」`.custom-input` `.rule-input` 是 inline 数字输入，
  不走 height: 48 这一档，保留 `padding: 6px 10px`
