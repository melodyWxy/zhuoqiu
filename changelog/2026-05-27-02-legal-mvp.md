# 隐私协议告知 + 微信直登 MVP

## 动机
微信小程序提交审核反馈：手机号属个人信息，必须先有《用户服务协议》和《隐私政策》明确告知收集目的、方式、用途，并取得授权同意才能获取。同时按《微信小程序用户隐私保护指引》，调用 `wx.login` 等隐私 API 前必须先经用户对协议的明示同意。

用户进一步澄清：在微信环境点登录、隐私授权通过后，**直接获取用户信息登录即可**，不再展示 actionsheet 菜单；H5 下因为没有真实微信通道，菜单只剩"手机号登录"。

## 遗留问题 / 已知限制
- 协议正文是 MVP 中文版，业务方名称、备案号、客服联系方式等字段使用占位（`support@jiqiubang.example`），上线前需要法务/运营复核与替换。
- 不接 Markdown 渲染，长协议如未来需要富文本（加粗、列表）需要另起方案。
- 不做"撤回授权"独立入口；当前用户撤回需要走"微信设置 → 隐私 → 授权管理"或注销账号。
- 服务端不记录同意时间戳；如审核要求强证据链，需加 `users.legal_agreed_at` 字段并在登录接口上报。
- 不开 `__usePrivacyCheck__` 全局拦截，避免影响除登录之外的接口；如未来要用 `wx.getUserProfile` 等其他隐私 API，需要再次评估是否拉起。
