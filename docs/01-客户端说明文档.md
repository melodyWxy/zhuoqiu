# 桌球计分 · 客户端（C 端 H5）说明文档

> **版本：** v2.20
> **适用对象：** 球友、参赛玩家、围观球迷
> **访问地址：** `https://billiards-client.macrobit.com.cn`（生产）
> **技术栈：** Taro · React · H5

---

## 1. 产品定位

桌球计分客户端（C 端）是给**普通球友**用的轻量 H5：你打一局九球或中八，掏出手机扫一下，就能跟对手实时同步比分、回看历史、参加球房办的赛事，不用注册一堆破账号也不用装 App。

**核心场景**：

- 朋友局：自己一个人记分太累，开个房间让对手扫码进来，俩人在自己手机上各自加分。
- 球房比赛：到球房参加擂台赛，扫赛事码报名 → 看自己被分到哪桌 → 上手就开始打。
- 看自己的战绩：今年打了多少局、谁是宿敌。

## 2. 主要功能

### 2.1 创建/加入比赛

- **创建比赛**：选模式（九球 / 中式八球）、设置规则（让球？盘数？），生成房间码，对手扫码加入。
- **加入比赛**：扫码或输入 6 位房间码进入对手开的房间。
- **联机计分**：双方任一人加分，另一边实时看到（基于 WebSocket）。断线重连不丢分。

### 2.2 球房 & 赛事

- **球房列表**（一级 Tab "球房"）：看附近 / 全部已入驻球房，点进店主页可看店铺信息、当前在办的赛事。
- **赛事列表**（一级 Tab "赛事"）：浏览所有正在报名 / 进行中的赛事，按球房 / 时间筛选。
- **赛事详情**：赛事信息（时间、奖品、报名费）+ 当前赛程树（bracket）+ 自己的对阵。
- **报名**：报名期内一键报名，填昵称 + 手机号（手机号给商家现场联系）。
- **开打入口**：赛事开始后，"我的对阵"里点对应轮次 → 直接进九球 / 中八的联机计分页。

### 2.3 我的

- 个人战绩、历史比赛、参与过的赛事。
- 切换登录方式（手机号验证码 / 微信）。
- **🏢 切换球房管理模式**：长按 / 点这里跳到商家登录页（详见《球房商家端说明文档》）。

### 2.4 商家身份在 C 端的"只读视角"

如果你是球房老板，可以用商家账号在 C 端登录，看到的是**只读**视角：

- 底部 Tab 变成 `首页 / 我的球房 / 我的赛事 / 我的`
- 能看到自家球房资料、自家赛事进度、玩家名单（脱敏）
- **不能**创建赛事 / 改店铺资料 / 开局 → 这些动作会提示"请到管理后台" + 跳转链接

理由：让老板在外面用手机也能盯进度，但创建动作集中在后台避免误操作。

## 3. 端到端登录与权限

| 身份 | 登录方式 | 能做什么 |
|---|---|---|
| **未登录访客** | 直接打开就能看 | 看球房 / 赛事列表，不能报名、不能开局 |
| **C 端用户** | 手机号 + 验证码（dev 期固定码 `123456`），微信登录待接 | 创建/加入比赛、报名赛事、看战绩 |
| **商家账号在 C 端登录** | 同手机号验证码（独立账号体系） | 仅只读看自家球房/赛事；写入操作引导跳后台 |

> 注：dev 期短信验证码固定为 `.env` 里的 `DEV_FIXED_SMS_CODE`，正式上线前务必清空。

## 4. 比赛核心规则（计分相关）

### 4.1 九球（Nine-ball）

- 进 1 号球得 1 分，进 9 号球直接获胜该局
- "小杰克"（一杆清）按规则配置可加分
- 让球：让分（开局送对手 N 分）/ 让球数（让对手先进 N 颗指定球）

### 4.2 中式八球（Chinese Eight-ball）

- 选半色 / 全色后，先把己方目标球全部进袋再打 8 号球
- 8 号球进则胜负判定（按规则）

### 4.3 联机协议

- 双方各自加分，事件走 WebSocket 同步，乐观 UI（先本地变，再确认）
- 断线重连：用 match_id 重新订阅，服务端重发最近事件做对齐

## 5. 端到端典型流程

### 5.1 朋友局

1. A 创建九球比赛 → 拿到房间码 `ABC123`
2. B 扫码 / 输码加入
3. 双方在各自手机上加分 → 实时同步
4. 比赛结束自动归档到双方"我的战绩"

### 5.2 报名球房擂台赛

1. C 端用户进入"赛事" Tab → 找到"五一擂台赛" → 点详情
2. 报名期内点 [报名] → 填昵称 / 手机号 → 提交
3. 商家在后台"开赛"后，bracket 生成
4. 用户在赛事详情看到"我的对阵 R1：vs 张三 · #5 桌"
5. 到球房现场，点对阵 → 进九球联机页 → 跟对手实时记分
6. 比赛结束 → 系统自动把 winner 写回 bracket → 进入下一轮 / 结束

## 6. 兼容性 & 性能

- 浏览器：iOS Safari 14+ / Android Chrome 90+ / 微信内置浏览器
- 弱网友好：所有写入走幂等接口，断线重连不丢分
- 包大小：H5 主包目标 < 500KB（gzipped），首屏 < 1.5s（4G）

## 7. 已知限制 & 路线图

- ❌ 暂无附近球房（LBS）：球房列表是"全部已入驻"按更新时间排序
- ❌ 暂无支付：报名费 / 奖金全部线下，平台不经手钱
- ❌ 暂无微信小程序版：仅 H5（含微信内打开）
- ⏳ v2.11+：排行榜 C 端展示、OSS 直传图片、真实短信通道
- ⏳ v2.20：API 域名拆分（前端调 `billiards-server.macrobit.com.cn/v1`）已完成
