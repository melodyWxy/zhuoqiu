# 桌球计分 · 球房商家端说明文档

> **版本：** v2.20
> **适用对象：** 球房老板（owner）、店员（staff，v2.11+）
> **访问地址：** `https://billiards-admin.macrobit.com.cn`（生产）
> **技术栈：** React · Vite · admin-web

---

## 1. 你能用它做什么

球房商家端是给**球房老板**用的运营后台，三件事：

1. **入驻**：交营业执照 → 平台审核通过 → 拿到自己的球房页面
2. **办赛事**：发布比赛 → 收报名 → 到点开赛 → 现场控台一桌一桌点开局 → 系统自动记冠军
3. **管店**：维护店铺资料、看自家所有比赛和报名情况

商家端**不收钱**：所有报名费、奖金都在线下处理，平台只负责"信息和荣誉系统"。

---

## 2. 入驻流程

### 2.1 两个入口

**A. C 端入口**（手机操作）

打开客户端 → 我的 → 🏢 切换球房管理模式 → 球房管理模式登录页 → [申请球房入驻]

**B. 后台入口**（电脑操作）

直接访问 `billiards-admin.macrobit.com.cn/apply` → 手机号注册商家账号 → 填申请表

> 两个入口落到同一张表，不区分来源，只记录 `source` 字段用于分析。

### 2.2 申请表内容

| 必填项 | 说明 |
|---|---|
| 店名 | 搜索键，注意品牌一致性 |
| 地址 | 文字地址 |
| 联系电话 | 客户找你用 |
| 台桌数 | 整数；本期不到台桌 ID 维度 |
| 营业执照图 | OSS 上传，用于平台审核 |
| 营业时间 | JSON：`{mon: "10:00-02:00", ...}` |
| 店铺介绍 | 富文本（可选） |
| 封面图 | 可选 |

提交后状态：`draft → pending`，等平台审核。

### 2.3 审核结果

- **通过**：自动创建 venue + 绑定你为 owner，发短信和站内信通知 → 进入"管理后台正式版"
- **驳回**：状态回到 `draft` + 显示驳回原因，你可以改完重新提交

### 2.4 后门：平台直接建房

合作球房 / 紧急情况下，平台运营可以在 admin 里直接建房：填店铺信息 + 老板手机号 → 跳过审核流程直接生成 venue。所有此类操作走审计日志。

## 3. 后台菜单

商家账号（`venue_account`）登录 admin 后看到：

```text
├─ 店铺主页              ← 公开展示给 C 端用户
├─ 店铺资料              ← 编辑店铺信息（地址、台桌数、营业时间、介绍）
├─ 赛事管理
│   ├─ 赛事列表（草稿 / 报名中 / 进行中 / 已结束）
│   ├─ + 新建赛事
│   └─ 赛事详情 → 报名列表 / bracket 树 / 现场控台
├─ 现场控台              ← 进行中比赛的快速入口
└─ 团队（占位，v2.11+）
```

## 4. 办一场赛事的完整流程

### 4.1 新建赛事（draft 阶段）

进入 **赛事管理 → + 新建赛事**，填：

| 字段 | 说明 |
|---|---|
| 赛事标题 | "五一九球擂台" |
| 项目 | 九球 / 中式八球 |
| 赛制 | 单败淘汰 ✅ / 其他赛制（v2.11+，先占位） |
| 报名上限 | 8 / 16 / 32 / 64 |
| 启动下限 | 默认 4，少于则不开赛 |
| 报名费（仅记录） | 整数（分） |
| 奖品文案 | 自由填："冠军 500 元 + 店内 3 小时券" |
| 报名时间窗 | 起 / 止 |
| 比赛开始时间 | 到点系统催你"开赛" |
| 规则配置 | 九球：让球 / 让分 / 小杰克加分等；中八：8 号球处理等 |
| 封面图 | 用于赛事列表卡片 |

保存为 `draft`，可以反复改。

### 4.2 发布（draft → registering）

点 [发布]，赛事进入 `registering` 状态：

- C 端"赛事"列表能看到、能报名
- 你能在赛事详情看到实时报名列表

### 4.3 开赛（registering → in_progress）

报名截止时间到 / 你手动点 [开赛]：

- 系统按报名顺序作为 `seed`，**一次性生成全部空 bracket**
- 人数不是 2^n（如 6 人）会自动给头部种子补 BYE 到 8
- 所有用户在赛事详情能看到"我的对阵"

### 4.4 现场控台（核心操作页）

赛事进行时，控台展示当前轮 bracket 树。每个 ready 状态的对阵会有一个 [开始比赛] 按钮：

```text
R1 (8 进 4)
├─ #1 张三 [BYE]                 → 自动晋级
├─ #4 李四 vs #5 王五   [开始]   ← 点这里
├─ #3 赵六 vs #6 钱七   [开始]
└─ #2 孙八 [BYE]
```

**点 [开始比赛]** 会做几件事：

1. 创建一条 `matches` 行（带 venue_id + bracket_match_id）
2. 通知双方 C 端"你的对阵已开始"
3. 双方在 C 端进入对应的九球 / 中八联机计分页

### 4.5 比赛结束自动推进 bracket

任一方点结束 → match.end 触发：

- 服务端把 winner 写回 `tournament_bracket_match.winner_registration_id`
- 自动生成（如果不存在）下一轮的对阵
- 当所有轮次完成 → 赛事 `completed` + 推送冠军通知

### 4.6 弃权 / Walkover

控台对每个 ready 对阵都有 [标记弃权] 按钮：选择获胜方 → bracket 直接推进，**不创建 match 行**。

### 4.7 取消赛事

`draft / registering` 状态可取消（发短信给已报名玩家退报名费）。`in_progress` 不能取消，只能让它打完或人工 walkover 到底。

## 5. 你能（不能）做什么

| 能力 | owner | staff（v2.11+） |
|---|:---:|:---:|
| 改店铺资料 | ✅ | ❌ |
| 发赛事 / 编辑赛事 | ✅ | ✅ |
| 查报名手机号 | ✅ | ✅ |
| 现场开局 / walkover | ✅ | ✅ |
| 加 / 删店员 | ✅ | ❌ |
| 看自家全部数据 | ✅ | ✅ |
| 看其他球房数据 | ❌ | ❌ |

> v2.10 只支持 owner 一人，UI 有"团队"占位入口（"即将上线"）。

## 6. 商家账号在 C 端的只读视角

同一套商家账号也能在 C 端登录，看到的是**只读**视角，详见《客户端说明文档》第 2.4 节。简单说：

- 能：看自家球房 / 历史赛事 / 当前赛事进度 / 玩家名单
- 不能：创建任何东西（点了会提示"请到管理后台"）

理由：让你出门在外用手机也能盯比赛进度，避免误操作。

---

## 7. 数据归属与排行榜

从 v2.10 起，**任何在球房场景下发起的比赛**都会写 `matches.venue_id`：

- 通过赛事控台开的对局：自动带 venue_id
- 商家自建"店内随手局"（v2.11）：要求选 venue
- C 端用户自己开的私局（不在球房）：venue_id = null

排行榜（日 / 周 / 月）的写入侧已就绪，**C 端展示放 v2.11**——届时不需要你做任何操作，球房页面会自动出现"本周本店九球王"等榜单。

## 8. 常见问题

**Q: 报名人数不是 2 的幂次怎么办？**
A: 系统自动给头部种子补 BYE，比如 6 人补到 8（2 个 BYE）。

**Q: 用户报名后可以退吗？**
A: 报名期内可以自助退报名。报名截止后只能联系商家在后台标 `withdrawn`，bracket 已生成的话该位置变 walkover。

**Q: 一个用户能同时报多场赛事吗？**
A: 能。时间冲突自理，平台不强阻挡。

**Q: 店员系统什么时候上？**
A: v2.11+。当前 owner 一个人扛。

**Q: 怎么改店铺地址？**
A: 入驻通过后在"店铺资料"里直接改，无需重新审核（除营业执照外）。

**Q: 营业执照过期了怎么办？**
A: 上传新执照 → 重新走审核（v2.11+ 实现，目前先联系平台运营）。
